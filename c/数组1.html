C语言数组与内存复习笔记

复习定位本笔记整理第 1 到第 6 节：数组概念、数组定义、数组初始化、数组元素访问、数组遍历、内存与内存地址。

目标不是背概念，而是能理解：数组为什么能存多个数据，为什么用下标访问，为什么下标从 0 开始，以及数组和内存地址之间的关系。

一、什么是数组

数组是一个容器，可以用来存储同种数据类型的多个值。

如果只存一个数据，用变量即可；如果要存很多个同类型数据，用很多变量会很麻烦，这时就应该使用数组。

int score = 90; // 一个变量只能存一个值

int scores[5] = {90, 85, 76, 92, 88}; // 一个数组可以存多个值

数组的核心作用

数组可以用一个名字管理一组数据。比如 5 个学生的成绩，可以用 scores 这个数组统一管理，而不是写 score1、score2、score3。

注意数组通常要求存储同一种数据类型。

比如 int 数组主要存整数，double 数组主要存小数。

int nums[3] = {10, 20, 30};
double prices[3] = {9.9, 15.5, 20.0};

关于隐式转换

C 语言中存在隐式转换。比如 double 数组可以存整数，因为整数会自动转成小数。

double arr[3] = {1, 2, 3}; 
// 实际理解为：1.0, 2.0, 3.0

易错如果把小数放进 int 数组，小数部分可能会丢失。

int arr[3] = {1.5, 2.8, 3.9};
// 可能变成：1, 2, 3

本节重点：数组就是用一个名字，管理一组相同类型的数据。

二、数组的定义

数组定义格式：

数据类型 数组名[数组长度];

例如：

int arr[3];

这句话表示：定义了一个名叫 arr 的数组，它最多可以存储 3 个 int 类型的整数。

组成部分 含义
int 数组中存放的数据类型是整数
arr 数组名，以后访问和修改数组都要用它
[3] 数组长度，表示最多存 3 个数据

区分int arr; 是普通变量，不是数组。

int arr[3]; 才是数组，因为后面有方括号和长度。

数组的两个重要特点

内存特点数组在内存中是一块连续空间。长度是多少，就有多少个连续的小格子。

长度固定普通数组一旦定义，长度不能改变。

例如 int arr[3]; 定义后，它只能存 3 个整数，不能自动变长，也不能自动变短。

定义数组练习

// 存储全班80位同学的年龄，年龄是整数
int ages[80];

// 存储全班50位同学的身高，身高可能是小数
double heights[50];

// 存储3件衣服的价格，价格可能是小数
double prices[3];

本节重点：定义数组，就是在内存中创建一块连续空间，用来存放多个相同类型的数据。

三、数组的初始化

初始化：定义数组的时候，第一次给数组赋值。

int ages[5] = {17, 18, 19, 20, 21};

左边是数组定义，右边大括号中是要放进数组的数据。数据之间要用英文逗号隔开。

基本格式

数据类型 数组名[数组长度] = {数据1, 数据2, 数据3};

规则 1：可以省略数组长度

如果初始化时省略数组长度，那么右边有几个数据，数组长度就是几。

int ages[] = {17, 18, 19, 20, 21};
// 数组长度自动确定为5

规则 2：写了长度时，数据个数不能超过长度

int nums[5] = {1, 2, 3}; // 正确，数据个数小于长度

int nums2[2] = {1, 2, 3}; // 错误，数据个数超过长度

规则 3：没给满的位置会补默认值

int nums[5] = {1, 2, 3};
// 实际效果：1, 2, 3, 0, 0

double arr[3] = {1.5};
// 实际效果：1.5, 0.0, 0.0
类型 未赋值时的默认补值
int 0
double 0.0
char '\0'，空字符

补充纠正字符串相关的空值概念后面会专门学习。当前阶段先记住：字符数组未填的位置常补 '\0'。
