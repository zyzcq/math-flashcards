C语言算术运算符复习笔记

覆盖：加减乘除取余、数值拆分、隐式转换、强制转换、字符相加、常见细节。

建议复习顺序：先看黄色重点，再看红色易错，最后跑代码。

一、运算符与算术运算符

变量能保存数据，运算符让数据发生计算、比较、判断等操作。

C语言常见运算符类别：算术运算符、自增自减运算符、赋值运算符、关系运算符、逻辑运算符、三元运算符。
运算 符号 说明
加法 + 求和
减法 - 求差
乘法 * C语言中不用 ×
除法 / 获取商；键盘上是斜杠
取余 % 获取余数；一般用 Shift + 5 输入

理解10 / 3 获取商，结果是 3；10 % 3 获取余数，结果是 1。

本部分重点

算术运算符一共五个：+、-、*、/、%。其中 / 要商，% 要余数。

二、整数、小数参与计算的规则

整数计算

整数和整数计算，结果仍然是整数。

10 / 3 的结果是 3，小数部分直接舍弃。

小数计算

只要有小数参与，结果通常就是小数。

10 / 3.0 的结果是 3.333333。

#include <stdio.h>

int main()
{
    printf("%d\n", 10 / 3);       // 3
    printf("%lf\n", 10 / 3.0);    // 3.333333
    printf("%.2lf\n", 10 / 3.0);  // 3.33

    return 0;
}
小数直接参与计算时，结果可能不精确。例如 10.0 / 3.0 显示很多位小数时，后面可能出现误差。这和小数在计算机中的存储方式有关。

本部分重点

想得到小数结果，必须让至少一个小数参与计算。只改输出格式不能改变计算结果。

三、除法和取余的易错点

规则 示例 说明
不能除以 0 10 / 0 错误
不能对 0 取余 10 % 0 错误
取余两边必须是整数 10.0 % 3 错误
余数正负看第一个数 -10 % 3 结果是 -1 
#include <stdio.h>

int main()
{
    printf("%d\n", 10 % 3);    // 1
    printf("%d\n", 10 % -3);   // 1
    printf("%d\n", -10 % 
    printf("%d\n", -10 % 3);   // -1
    printf("%d\n", -10 % -3);  // -1

    return 0;
}
易错：% 只能用于整数，不能写 10.0 % 3 或 10 % 3.0。

本部分重点

/ 和 % 都和除法有关：/ 得商，% 得余数。除数不能是 0，取余两边必须是整数。

四、练习：三位数拆分

需求：键盘录入一个三位数，把它拆成个位、十位、百位并输出。

核心公式：先用 / 去掉右边不要的数字，再用 % 10 获取当前最后一位。
目标 公式 以 123 为例
个位 number % 10 123 % 10 = 3
十位 number / 10 % 10 123 / 10 = 12，12 % 10 = 2
百位 number / 100 % 10 123 / 100 = 1，1 % 10 = 1 
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
    int number;

    printf("请输入一个三位数：\n");
    scanf("%d", &number);

    int ge = number % 10;
    int shi = number / 10 % 10;
    int bai = number / 100 % 10;

    printf("个位是：%d\n", ge);
    printf("十位是：%d\n", shi);
    printf("百位是：%d\n", bai);

    return 0;
}
好习惯：写代码时建议“写一部分，测一部分”。比如先测试 scanf 是否能成功接收输入，再写拆分逻辑。

本部分重点

拆数字的通用公式是：某一位 = number / 位权 % 10。例如千位是 number / 1000 % 10。

五、隐式转换：小类型自动转大类型

不同数据类型参与计算时，C语言通常会先把它们转成同一种类型，再进行计算。

隐式转换：取值范围小的数据类型，自动转换成取值范围大的数据类型。不需要手写代码。

常见取值范围关系可以先按下面顺序记：

double > float > long long > long > int > short > char

规则一：小范围类型 + 大范围类型，小的转成大的

#include <stdio.h>

int main()
{
    int a = 10;
    double b = 3.14;

    double c = a + b;

    printf("%.2lf\n", c);  // 13.14

    return 0;
}

分析：a 是 int，b 是 double。计算时，int 会先转成 double，所以结果是 double。

规则二：short 和 char 运算前会先提升为 int

#include <stdio.h>

int main()
{
    short s1 = 10;
    short s2 = 20;

    int result = s1 + s2;

    printf("%d\n", result);  // 30

    return 0;
}

两个 short 相加，结果不是 short，而是 int。

易错：不要以为“两个什么类型相加，结果就是什么类型”。short + short 的结果是 int。

本部分重点

隐式转换有两个核心：小类型转大类型；short 和 char 运算前会先转成 int。

六、强制转换：大类型手动转小类型

如果要把取值范围大的数据赋值给取值范围小的变量，就需要强制转换。

强制转换格式：(目标类型)数据。想转成什么类型，就在前面小括号里写什么类型。 
#include <stdio.h>

int main()
{
    int a = 10;
    short b = (short)a;

    printf("%hd\n", b);

    return 0;
}

强制转换的底层可以简单理解为：大空间的数据塞进小空间里，多余部分可能被丢掉。

风险：强制转换可能导致数据错误。例如把较大的 int 强制转成 short，如果超出 short 的范围，结果可能不是原来的值。

short + short 的正确强转

#include <stdio.h>

int main()
{
    short s1 = 10;
    short s2 = 20;

    short result = (short)(s1 + s2);

    printf("%hd\n", result);

    return 0;
}

重点是这行：

short result = (short)(s1 + s2);

它表示先计算 s1 + s2，再把整体结果转成 short。

错误理解：(short)s1 + s2 只是转换了 s1，没有转换整个表达式结果。要强转表达式结果，必须写 (short)(s1 + s2)。

本部分重点

强制转换用于“大转小”，必须手动写。格式是 (目标类型)数据。强转表达式结果时，要用括号把表达式整体括起来。

七、字符相加：char 如何参与计算

char 是字符类型，但它可以参与加法运算，因为字符底层会对应一个数字编码。

字符和数字之间的转换依靠 ASCII 码表。字符参与运算时，会先提升为 int。
字符 ASCII 十进制值 说明
'a' 97 小写字母 a
'A' 65 大写字母 A
'0' 48 字符零，不是数字 0 
#include <stdio.h>

int main()
{
    char c = 'a';
    int i = c + 1;

    printf("%d\n", i);  // 98

    return 0;
}

分析：'a' 对应 97，所以 'a' + 1 实际是 97 + 1，结果是 98。

数字和字符的两种打印方式

#include <stdio.h>

int main()
{
    char c = 97;

    printf("%d\n", c);  // 97，按整数打印
    printf("%c\n", c);  // a，按字符打印

    return 0;
}

同一个数据，用 %d 打印是整数，用 %c 打印是字符。

不用死背 ASCII，用代码查

#include <stdio.h>

int main()
{
    printf("%d\n", 'a');  // 97
    printf("%d\n", 'A');  // 65
    printf("%d\n", '0');  // 48

    return 0;
}

本部分重点

char 参与运算时会查 ASCII 码并提升为 int。'a' 是 97，'A' 是 65，'0' 是 48。

八、格式控制符复习

格式 用途 示例
%d 输出整数 printf("%d\n", 10);
%lf 输出 double 小数 printf("%lf\n", 3.14);
%.2lf 输出 double，保留两位小数 printf("%.2lf\n", 3.14159);
%c 输出字符 printf("%c\n", 'a');
%hd 输出 short printf("%hd\n", s);
%lld 输出 long long printf("%lld\n", n);
%zu 输出 sizeof 的结果 printf("%zu\n", sizeof(int));
%% 打印百分号本身 printf("10 %% 3");

本部分重点

输出什么类型，就尽量使用匹配的格式控制符。% 在 printf 中有特殊含义，想打印百分号本身要写 %%。

九、最终复习清单

  • 基础算术运算符有五个：+、-、*、/、%。
  • 区别/ 获取商，% 获取余数。
  • 易错整数和整数计算，结果仍然是整数，10 / 3 是 3。
  • 技巧想得到小数结果，至少让一个小数参与计算，如 10 / 3.0。
  • 禁止不能除以 0，也不能对 0 取余。
  • 取余% 两边必须都是整数。
  • 符号取余结果的正负号和第一个数保持一致。
  • 拆分个位：number % 10；十位：number / 10 % 10；百位：number / 100 % 10。
  • 隐式转换小范围类型会自动转成大范围类型。
  • 提升short 和 char 运算前会先提升为 int。
  • 强制转换格式是 (目标类型)数据。
  • 风险强制转换可能造成数据丢失或结果错误。
  • 字符字符参与运算时，会根据 ASCII 码转成整数。
  • ASCII'a' = 97，'A' = 65，'0' = 48。
  • 习惯写代码时建议“写一部分，测一部分”。

十、综合代码：一份代码复习全部重点

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
    // 1. 算术运算符
    printf("10 + 3 = %d\n", 10 + 3);
    printf("10 - 3 = %d\n", 10 - 3);
    printf("10 * 3 = %d\n", 10 * 3);
    printf("10 / 3 = %d\n", 10 / 3);
    printf("10 %% 3 = %d\n", 10 % 3);

    // 2. 整数除法与小数除法
    printf("整数除法：%d\n", 10 / 3);
    printf("小数除法：%.2lf\n", 10 / 3.0);

    // 3. 三位数拆分
    int number = 123;
    int ge = number % 10;
    int shi = number / 10 % 10;
    int bai = number / 100 % 10;

    printf("个位：%d\n", ge);
    printf("十位：%d\n", shi);
    printf("百位：%d\n", bai);

    // 4. 隐式转换
    int a = 10;
    double b = 3.14;
    double c = a + b;
    printf("int + double = %.2lf\n", c);

    // 5. short 运算提升为 int
    short s1 = 10;
    short s2 = 20;
    int r1 = s1 + s2;
    printf("short + short 结果是 int：%d\n", r1);

    // 6. 强制转换
    short r2 = (short)(s1 + s2);
    printf("强制转换为 short：%hd\n", r2);

    // 7. 字符相加
    char ch = 'a';
    int result = ch + 1;
    printf("'a' + 1 = %d\n", result);

    // 8. ASCII 查询
    printf("'a' 对应：%d\n", 'a');
    printf("'A' 对应：%d\n", 'A');
    printf("'0' 对应：%d\n", '0');

    return 0;
}

本部分重点

这份综合代码适合最后复习时跑一遍。你不只要看结果，更要能解释每一行为什么得到这个结果。
