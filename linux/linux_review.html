3. 目录结构速记

/etc：系统配置文件，如账号、网络、服务配置。

/home：普通用户主目录。

/var：经常变化的数据，如 /var/log 日志。

/dev：设备文件目录，体现“一切皆文件”。

总结构：Linux 没有 C/D 盘，是从 / 开始的单根树状目录，文件名区分大小写。

4. 用户、组与账号文件

项目答案
root UID0
/etc/passwd保存用户基本信息，不保存真实密码
/etc/shadow保存加密后的密码哈希和密码策略
用户与组一个用户有一个主组，可以有多个附加组；一个组可包含多个用户
系统用户常用于运行服务进程，通常禁止登录

5. 权限与链接

权限含义

权限对文件对目录
r查看文件内容列出目录文件名
w修改文件内容创建、删除、重命名目录内文件
x作为程序/脚本执行进入目录、访问下级路径

软硬链接

软链接保存目标路径，类似快捷方式，可跨文件系统，可指向目录；硬链接共享 inode，是原文件别名，不能跨文件系统，通常不能指向目录。删除原文件后，软链接失效，硬链接不受影响。

6. 文本处理三剑客

grep：搜索过滤文本行，例如 grep "Listen" app.conf。

sed：流编辑器，按行替换/删除/新增；默认不改原文件，加 -i 才原地修改。

awk：按列提取、统计、格式化输出，适合处理表格化文本。

cut：简单列截取，常作为 awk 的轻量替代。

9. Shell 脚本模板

检测目录

#!/bin/bash
read -p "请输入要检测的目录路径: " dir_path
if [ -d "$dir_path" ]; then
    echo "目录存在，文件有："
    ls "$dir_path"
else
    echo "提示：该目录不存在！"
fi

检查文件

#!/bin/bash
read -p "请输入文件名字: " file_name
if [ -f "$file_name" ]; then
    echo "文件存在，内容如下："
    cat "$file_name"
else
    echo "文件不存在"
fi

1 到 20 求和

#!/bin/bash
sum=0
for (( i=1; i<=20; i++ ))
do
    sum=$((sum + i))
done
echo "1~20的整数和为: $sum"

1 到 20 奇数和

#!/bin/bash
sum=0
for (( i=1; i<=20; i+=2 ))
do
    sum=$((sum + i))
done
echo "1~20之间的奇数和为: $sum"

10. 考前 30 分钟清单

  • /etc/passwd 不存真实密码，/etc/shadow 存密码哈希。
  • root 的 UID 是 0。
  • 硬链接不能跨文件系统，软链接可以。
  • sed 默认不改原文件，加 -i 才原地改。
  • SSH 加密，Telnet 明文。
  • mount 挂载，umount 卸载。
  • Linux 是单根目录结构，没有 C/D 盘。
  • RAID0 提速无冗余，RAID1 镜像更安全。
  • 命令题必须自己能写：mkdir -p、tail -f、wc -l、chmod 744。
