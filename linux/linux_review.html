1. 系统基础与目录结构

🧠

特点与目录口诀

开多稳移，配置家园，变化设备

四大特点 (开多稳移)

开源免费、多用户多任务、稳定安全、可移植性强。

内核 (Kernel)

系统的核心，负责进程调度、内存管理、文件系统、设备驱动。

Shell

命令解释器，是用户和内核之间的桥梁。(注意：Shell 不是内核的一部分)

核心目录 (单根目录树 /)

目录作用说明记忆
/etc核心配置文件配置 (etc=everything to configure)
/home普通用户主目录家园 (注：超级管理员主目录在 /root)
/var变化的数据变化 (var=variable)，如 /var/log 存放日志
/dev设备文件设备 (dev=device)，体现Linux“一切皆文件”
易错提醒：
1. Linux 是从 / 开始的单根树状目录，绝对没有 C盘、D盘的概念。
2. 文件名严格区分大小写。
3. openEuler 是国产开源发行版，归属于 Linux 发行版大家族。

2. 用户、组与权限管理

🧠

账号与权限口诀

passwd明信息，shadow藏密码；
文件读写跑，目录看改进

账号文件区别

/etc/passwd 保存基本信息(用户名, UID, 登录Shell等)，绝对不存真实密码。
/etc/shadow 保存加密密码哈希，仅 root 可读，保护安全。

主组与附加组

用户创建文件时默认归属主组(且必须有一个主组)，还可以加入多个附加组获取额外权限。root 的 UID 永远是 0。

权限位计算与含义 (重点)

权限分三种：r (读) = 4, w (写) = 2, x (执行) = 1。没有权限就是 0 或 -。

权限组合数字对【文件】的含义对【目录】的含义
rwx7 (4+2+1)读写跑(执行)看(ls)改(创建/删除)进(cd)
rw-6 (4+2)读写，不能执行能看能改，但进不去(缺x不能cd)
r-x5 (4+1)只读和执行能看能进，不能在目录里新建/删除
r--4 (4+0+0)只读只能列出文件名，看不了详情也进不去

举一反三：执行 chmod 755 run.sh 意为：所有者拥有 rwx(7)，所属组拥有 r-x(5)，其他人拥有 r-x(5)。

3. 链接与文本三剑客

🧠

链接与文本口诀

软路硬号，跨区看软；搜改列，三剑客

软硬链接高频对比

特性软链接 (Soft/Symbolic Link)硬链接 (Hard Link)
底层原理保存目标路径的新文件 (快捷方式)同一个文件内容的不同文件名 (同inode号)
跨文件系统可以跨越 (推荐)不可以
指向目录可以通常不可以 (防止形成死循环)
原文件被删链接失效，红底闪烁 (变成死链接)不受影响，依然可以访问数据内容

文本三剑客 (必考)

grep (搜)

按关键字或正则表达式搜索过滤文本行。
例: grep "Error" log.txt 找报错行。

sed (改)

流编辑器，非交互式按行替换/删除/新增。
坑：默认不改原文件，加 -i 才原地修改！

awk (列)

按指定分隔符分割字段，适合处理表格列数据。
例: 提取第一列 awk '{print $1}'

补充知识：find 侧重于根据文件名、大小在磁盘找文件；grep 侧重于在文本内容中找字符串。

4. 进程、网络与安全

🧠

进程与安全口诀

程序静进程动，查PID，kill终止

进程基础管理

程序是磁盘上静态的代码，进程是内存中动态的运行实例。
查看：ps -ef (看全部快照), top (实时动态资源占用)
终止：kill PID (发终止信号), kill -9 PID (强杀)
挂起后台运行：命令末尾加 &

网络防御与安全

系统安全三件套：账号管理、权限控制、网络防御。
远程协议：SSH 是加密安全传输，目前绝对主流；Telnet 是明文传输，极容易被抓包，不安全。
看端口：netstat -tulnp 或 ss -tulnp 查监听。

5. 存储挂载、包管理与 RAID

RAID 阵列常考对比

记忆口诀：【0提速无备份，1镜像保安全】

类型工作原理优缺点与应用场景
RAID 0条带化 (Striping)读写极快，但无冗余，坏一块盘数据全毁。适合缓存、草稿盘。
RAID 1镜像备份 (Mirroring)安全性极高，但空间利用率仅 50%，成本高。适合系统盘、核心数据。
磁盘挂载 (Mount)

Linux必须把设备挂载到目录树节点才能访问。
挂载：mount /dev/sdb1 /data
卸载：umount /data (注意拼写没有字母n!)
常见文件系统：ext4 (本地系统), NFS (网络共享系统)。

两大软件包体系

红帽系 (CentOS/openEuler/RHEL):
底层包格式为 .rpm，高级依赖管理工具为 yum 或新一代 dnf。
Debian系 (Ubuntu/Kali):
底层包格式为 .deb，高级工具为 apt / apt-get。

6. 必写实操命令大全

⚠️ 提示：期末简答或填空题通常要求直接手写命令，特别注意参数含义及空格。

需求场景满分命令写法关键参数记忆法
显示所有文件详细信息ls -la-l (long)详细, -a (all)显示隐藏文件(点开头)
递归创建多层嵌套目录mkdir -p /data/log/web-p (parents) 父目录不存在时自动创建
复制文件到指定目录cp file1.txt /tmp/backup/cp 源文件 目标路径
移动文件 / 重命名mv a.txt b.txtmv 在同目录下即为重命名，不同目录即为移动
递归强制删除目录rm -rf /test-r (recursive)递归, -f (force)强制免确认，危险操作！
查看日志文件最后 15 行tail -n 15 /var/log/syslog-n (number) 指定查看的行数
动态实时追踪日志刷出tail -f /var/log/syslog-f (follow) 实时跟踪末尾变化，常用于查错
打包并压缩成 tar.gztar -czvf app.tar.gz ./app-c(创建) -z(gzip压缩) -v(显示过程) -f(指定文件名)
解压 tar.gz 压缩包tar -xzvf app.tar.gz-x(解压 extract)，其余同上
统计 passwd 文件有多少行wc -l /etc/passwd-l (lines) 仅统计行数
设置权限：所有者全权，其他读chmod 744 run.sh7(rwx), 4(r--), 4(r--)

Vim 编辑器核心快捷键 (高频考点)

Vim 有模式之分：打开默认是命令模式，按键被当作命令；按 i 等进入插入模式打字；任何时候按 ESC 回到命令模式。

切换进入插入模式

i : 在当前光标前插入。
o : 在当前行下方新开一行插入。
a : 在当前光标后追加。

保存与退出 (需在命令模式)

输入 :w 保存，:q 退出。
输入 :wq 或 :x 保存并退出。
输入 :q! 强制退出且不保存修改。

命令模式快捷编辑

yy 复制当前行，p 在下方粘贴。
dd 删除(剪切)当前行。
u 撤销上一步操作。

7. 核心 Shell 脚本默写模板

默写关键规则：
1. 第一行必须写 #!/bin/bash 指定解释器。
2. if 后的 [ ] 括号内侧，两边必须各有一个空格！(如 [ -f file ])。
3. 变量赋值时，等号两边绝对不能有空格！(如 sum=0，错写为 sum = 0 扣全分)。
模板 1：检测文件是否存在，存在则读取 (判断 -f)
#!/bin/bash
read -p "请输入文件名字: " file_name
if [ -f "$file_name" ]; then
    echo "文件存在，内容如下："
    cat "$file_name"
else
    echo "提示：文件不存在！"
fi
模板 2：检测目录是否存在，存在则查看 (判断 -d)
#!/bin/bash
read -p "请输入要检测的目录路径: " dir_path
if [ -d "$dir_path" ]; then
    echo "目录存在，里面的文件有："
    ls "$dir_path"
else
    echo "提示：该目录不存在！"
fi
模板 3：for 循环 1~20 整数求和

算术累加操作必须用 $(( )) 包裹起来计算。

#!/bin/bash
sum=0
for (( i=1; i<=20; i++ ))
do
    sum=$((sum + i))
done
echo "1~20的整数和为: $sum"
模板 4：1~20 奇数和 / 偶数和

奇数从1开始每次+2；若考偶数，只需改初值 i=2。

#!/bin/bash
sum=0
for (( i=1; i<=20; i+=2 )) 
do
    sum=$((sum + i))
done
echo "1~20奇数和为: $sum"
