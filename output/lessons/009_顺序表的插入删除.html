学习目标

  • 掌握顺序表插入操作的移动方向、边界检查和 length 更新。
  • 掌握顺序表删除操作的移动方向、删除值返回和 length 更新。
  • 能分析顺序表插入、删除的最好、最坏、平均时间复杂度。

知识点目录

  • 插入操作的基本思想：在顺序表第 i 个位置插入元素 e，需要先把第 i 个位置及其后面的元素依次后移一位，再把 e 放入下标 i-1 的位置，最后 length 加 1。

插入操作的基本思想

在顺序表第 i 个位置插入元素 e，需要先把第 i 个位置及其后面的元素依次后移一位，再把 e 放入下标 i-1 的位置，最后 length 加 1。

移动元素时必须从后往前移动。若从前往后移动，后面的旧元素会被覆盖，数据会丢失。插入前还要检查 i 是否在 1 到 length+1 之间，以及表是否已满。

关键提示
  • 插入第 i 位，数组下标是 i-1。
本部分重点：顺序表插入的关键是后移元素，从后往前移，最后维护 length。

删除操作的基本思想

删除第 i 个元素时，先把 data[i-1] 保存到输出参数 e 中，再把 i 之后的元素依次前移一位，最后 length 减 1。

删除操作要检查 i 是否在 1 到 length 之间。e 需要把删除的值带回调用者，在 C 语言中应使用 ElemType *e。顺序表本身会被修改，因此也应传入 SqList *L。

关键提示
  • 删除时移动方向与插入相反。
本部分重点：顺序表删除的关键是保存被删元素、从前往后前移元素、更新 length。

时间复杂度分析

插入最好情况是插入表尾，无需移动元素，时间复杂度 O(1)；最坏情况是插入表头，需要移动 n 个元素，时间复杂度 O(n)。若各插入位置等概率，平均需要移动 n/2 个元素，平均时间复杂度 O(n)。

删除最好情况是删除表尾，无需移动元素，时间复杂度 O(1)；最坏情况是删除表头，需要移动 n-1 个元素，时间复杂度 O(n)。平均时间复杂度同样为 O(n)。

关键提示
  • 表尾操作快，表头操作移动最多。
本部分重点：顺序表插删的主要时间开销来自移动元素，平均和最坏都是 O(n)。

健壮性和接口反馈

一个好的基本操作不能只在正确输入下工作，还要能处理非法位置、表满、空表等边界情况。函数返回 bool 值可以让调用者知道操作是否成功。

这些检查不是形式主义。边界条件正是程序最容易出错的地方，尤其是初学者写插入、删除循环时，移动范围和下标很容易写错。

关键提示
  • 返回 true/false 是清晰的失败反馈。
本部分重点：基本操作要让调用者用得明白，也要让错误输入被及时拒绝。

代码整理

顺序表插入与删除

使用 C 指针参数实现课程中的插入、删除逻辑。

顺序表插入与删除C
#include <stdbool.h>
#define MaxSize 50
typedef int ElemType;

typedef struct {
    ElemType data[MaxSize];
    int length;
} SqList;

bool ListInsert(SqList *L, int i, ElemType e) {
    if (i < 1 || i > L->length + 1) {
        return false;
    }
    if (L->length >= MaxSize) {
        return false;
    }
    for (int j = L->length; j >= i; --j) {
        L->data[j] = L->data[j - 1];
    }
    L->data[i - 1] = e;
    L->length++;
    return true;
}

bool ListDelete(SqList *L, int i, ElemType *e) {
    if (i < 1 || i > L->length) {
        return false;
    }
    *e = L->data[i - 1];
    for (int j = i; j < L->length; ++j) {
        L->data[j - 1] = L->data[j];
    }
    L->length--;
    return true;
}

易错点

插入移动方向写反：插入要从后往前移动，否则会覆盖尚未移动的数据。
忘记维护 length：插入后 length++，删除后 length--，否则表的逻辑长度错误。
位序和下标差 1：第 i 个元素对应 data[i-1]，循环边界要特别注意。
删除值没有带回：C 语言中应使用 ElemType *e 返回被删除元素。

本节总结

  • 顺序表插入需要后移元素，删除需要前移元素。
  • 插入合法位置是 1 到 length+1，删除合法位置是 1 到 length。
  • 插入和删除的最好时间复杂度为 O(1)，最坏和平均为 O(n)。
  • 健壮的代码要检查非法参数和容量状态，并通过返回值反馈成功或失败。

自测题

1. 顺序表插入时，移动元素应从哪里开始？
答案：B。从后向前移动才能避免覆盖旧数据。
2. 在长度为 n 的顺序表表头插入元素，需要移动多少个元素？
答案：C。原有 n 个元素都要后移一位。
3. 删除第 i 个元素后，length 应如何变化？
答案：C。删除一个元素后当前长度减少 1。
