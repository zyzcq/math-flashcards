按位查找与随机存取

按位查找是取得线性表第 i 个元素。顺序表中元素连续存放且大小相同，因此第 i 个元素可直接用 data[i-1] 访问，时间复杂度为 O(1)。

无论是静态数组还是动态分配的 data 指针，都可以使用下标形式访问。编译器会根据指针指向类型的大小，计算 data + 下标 对应的真实地址。

关键提示
  • 指针类型会影响每个下标跨过多少字节。
本部分重点：顺序表的按位查找不需要循环，体现了随机存取特性。

按值查找的实现

按值查找是在顺序表中寻找值等于 e 的元素，并返回其位序。若顺序表无序，只能从第一个元素开始依次比较，找到后返回 i+1，找不到返回 0 或其他约定失败值。

如果目标在第一个位置，最好时间复杂度 O(1)；如果在最后或不存在，最坏时间复杂度 O(n)；等概率出现时，平均时间复杂度 O(n)。

关键提示
  • 返回位序时要把数组下标加 1。
本部分重点：无序顺序表按值查找需要顺序扫描，平均和最坏都是 O(n)。

结构体比较的注意点

对于 int、char、double、float 等基本类型，可以直接使用 == 比较。对于 struct 结构体类型，C 语言不能直接用 == 判断两个结构体是否相等。

如果数据元素是结构体，应根据业务需求逐个比较关键字段，或者封装一个 equal 函数。数据结构考试有时允许抽象地写 e1 == e2，但若题目要求严格 C 语言代码，就要写出字段比较。

关键提示
  • 考试时要看题目是否要求严格 C 语法。
本部分重点：结构体按值查找要明确“相等”的业务含义，C 代码中不能直接用 == 比较结构体。

代码整理

顺序表按位与按值查找

按位查找使用 data[i-1]，按值查找顺序扫描。

顺序表按位与按值查找C
#include <stdbool.h>
#define MaxSize 50
typedef int ElemType;

typedef struct {
    ElemType data[MaxSize];
    int length;
} SqList;

bool GetElem(const SqList *L, int i, ElemType *e) {
    if (i < 1 || i > L->length) {
        return false;
    }
    *e = L->data[i - 1];
    return true;
}

int LocateElem(const SqList *L, ElemType e) {
    for (int i = 0; i < L->length; ++i) {
        if (L->data[i] == e) {
            return i + 1;
        }
    }
    return 0;
}

结构体相等判断

当 ElemType 是结构体时，需要自己定义比较逻辑。

结构体相等判断C
#include <stdbool.h>

typedef struct {
    int ticketNo;
    int people;
} Customer;

bool CustomerEqual(Customer a, Customer b) {
    return a.ticketNo == b.ticketNo && a.people == b.people;
}

易错点

按位查找返回 data[i]：位序 i 对应下标 i-1，直接写 data[i] 会错一位。
认为指针下标总是移动 1 字节：p[i] 的地址步长由 p 指向的数据类型大小决定。
直接用 == 比较结构体：标准 C 不支持结构体直接相等比较，需要逐字段判断。

本节总结

  • 顺序表按位查找 O(1)，因为可根据地址公式直接定位。
  • 无序顺序表按值查找需要顺序扫描，复杂度通常为 O(n)。
  • 基本类型可用 ==，结构体要自己定义相等规则。

自测题

1. 顺序表按位查找的时间复杂度是？
答案：A。连续存储且元素大小相同，可直接定位。
2. LocateElem 返回位序时，找到下标 i 的元素应返回？
答案：C。位序从 1 开始，下标从 0 开始。
3. C 语言中两个 struct 变量能否直接用 == 比较？
答案：B。标准 C 不支持结构体直接相等比较。
