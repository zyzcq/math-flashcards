链式存储与单链表结点

单链表是用链式存储实现的线性表。每个结点包含数据域 data 和指针域 next，next 指向下一个结点。由于每个结点只有一个后继指针，所以称为单链表。

链表中的结点可以离散存放在内存各处，拓展容量时只需申请一个新结点并修改指针，不要求整片连续空间。但单链表不支持随机存取，查找第 i 个结点只能从头开始沿 next 逐个向后找。

关键提示
  • 结点 = 数据域 + 指针域。
本部分重点：单链表用指针表示前后关系，换来了容量灵活性，也失去了随机存取。

C 语言中的结点定义与 typedef

可以用 struct LNode 定义结点类型，其中 data 保存数据元素，struct LNode *next 保存后继结点地址。typedef 可为类型取别名，让代码更简洁。

LNode * 强调“这是一个结点指针”；LinkList 本质上也是 LNode *，但语义上强调“这是一个链表的头指针”。教材中两种写法都会出现，关键是理解它们底层等价、语义侧重点不同。

关键提示
  • typedef 是类型重命名，不会创建变量。
本部分重点：LNode * 和 LinkList 类型等价，但命名表达的语义不同。

带头结点与不带头结点

不带头结点时，头指针 L 直接指向第一个数据结点；空表时 L == NULL。初始化时需要把 L 设为 NULL，避免脏数据。

带头结点时，L 指向一个不存放实际数据的头结点；空表时 L->next == NULL。带头结点能把表头插入、删除等操作统一成普通位置操作，代码通常更简单，因此后续默认多采用带头结点写法。

关键提示
  • 两种实现的判空条件不同。
本部分重点：带头结点让表头操作更统一，不带头结点在处理第一个元素时常要特殊分支。

代码整理

带头结点单链表定义与初始化

整理 LNode、LinkList、InitList 和 Empty 的 C 写法。

带头结点单链表定义与初始化C
#include <stdbool.h>
#include <stdlib.h>

typedef int ElemType;

typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;

bool InitList(LinkList *L) {
    *L = (LNode *)malloc(sizeof(LNode));
    if (*L == NULL) {
        return false;
    }
    (*L)->next = NULL;
    return true;
}

bool Empty(LinkList L) {
    return L->next == NULL;
}

不带头结点的初始化与判空

不带头结点时，头指针直接代表第一个数据结点。

不带头结点的初始化与判空C
#include <stdbool.h>

typedef int ElemType;
typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;

void InitListWithoutHead(LinkList *L) {
    *L = NULL;
}

bool EmptyWithoutHead(LinkList L) {
    return L == NULL;
}

易错点

把 LinkList 当成新结构体：LinkList 通常只是 LNode * 的别名，表示链表头指针。
判空条件写错：带头结点看 L->next，不带头结点看 L 本身是否为 NULL。
忘记初始化头指针或头结点 next：未初始化的指针可能含有脏数据，导致访问非法地址。

本节总结

  • 单链表结点由 data 和 next 组成，next 表示逻辑后继。
  • 链式存储不要求物理连续，容量扩展方便，但不支持随机存取。
  • 带头结点和不带头结点的初始化、判空与表头操作处理方式不同。

自测题

1. 单链表结点通常包含哪两部分？
答案：A。单链表结点包含数据域和后继指针域。
2. 带头结点单链表为空时，通常满足什么条件？
答案：B。头结点存在，但没有第一个数据结点。
3. 单链表为什么不能随机存取？
答案：B。必须从头结点沿指针逐个定位目标结点。
