# 2.3.2-3 单链表的建立

尾插法、头插法与链表逆置思想

## 学习目标

- 掌握尾插法建立单链表，并理解尾指针 r 的作用。
- 掌握头插法建立单链表，并理解输入序列会被逆置。
- 理解头插法可用于链表逆置这一常见考点。

## 知识点结构

- **尾插法建立单链表：** 尾插法每次把新结点插入表尾。若每次都从头查找尾结点，建立 n 个结点需要 0+1+...+(n-1) 次查找，复杂度 O(n^2)。
- **头插法建立单链表：** 头插法每次把新结点插入头结点之后。核心代码是 s->next = L->next; L->next = s。因为每次都插到表头，输入顺序为 10、16、27 时...
- **头插法与链表逆置：** 因为头插法会逆置输入顺序，所以常用于链表逆置。可以新建一个空链表，依次取旧链表元素并头插到新链表；也可以把旧链表的结点逐个摘下，再头插回原头结点之后，实现原地逆...

## 一、尾插法建立单链表

尾插法每次把新结点插入表尾。若每次都从头查找尾结点，建立 n 个结点需要 0+1+...+(n-1) 次查找，复杂度 O(n^2)。

更好的做法是维护尾指针 r，让 r 始终指向当前尾结点。每读入一个数据元素，就申请新结点 s，令 r->next = s，再令 r = s。最终把 r->next 置为 NULL。这样建立 n 个结点只需 O(n)。

关键提示：
- r 永远指向当前最后一个数据结点。

**本部分重点：** 尾插法要用尾指针 r 避免每次从头找表尾。


## 二、头插法建立单链表

头插法每次把新结点插入头结点之后。核心代码是 s->next = L->next; L->next = s。因为每次都插到表头，输入顺序为 10、16、27 时，最终链表顺序会变成 27、16、10。

初始化时应把头结点的 next 置为 NULL。若不初始化，头结点 next 可能含有脏数据，导致最后一个结点指向未知区域。

关键提示：
- 头插法的链表方向与输入顺序相反。

**本部分重点：** 头插法简单高效，但会得到输入序列的逆序。


## 三、头插法与链表逆置

因为头插法会逆置输入顺序，所以常用于链表逆置。可以新建一个空链表，依次取旧链表元素并头插到新链表；也可以把旧链表的结点逐个摘下，再头插回原头结点之后，实现原地逆置。

这一思想比死记代码更重要：很多复杂链表题都可以拆成“摘下一个结点”和“把结点头插到另一个位置”。

关键提示：
- 逆置链表时可以反复执行头插。

**本部分重点：** 头插法不仅用于建表，也是一类链表逆置题的核心思路。

## 代码整理

### 头插法与尾插法

以数组输入代替 scanf，便于阅读和测试。

```c
#include <stdbool.h>
#include <stdlib.h>

typedef int ElemType;
typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;

bool InitList(LinkList *L) {
    *L = (LNode *)malloc(sizeof(LNode));
    if (*L == NULL) {
        return false;
    }
    (*L)->next = NULL;
    return true;
}

bool CreateListTail(LinkList *L, const ElemType a[], int n) {
    if (!InitList(L)) {
        return false;
    }
    LNode *r = *L;
    for (int i = 0; i < n; ++i) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return false;
        }
        s->data = a[i];
        s->next = NULL;
        r->next = s;
        r = s;
    }
    return true;
}

bool CreateListHead(LinkList *L, const ElemType a[], int n) {
    if (!InitList(L)) {
        return false;
    }
    for (int i = 0; i < n; ++i) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return false;
        }
        s->data = a[i];
        s->next = (*L)->next;
        (*L)->next = s;
    }
    return true;
}
```

## 易错点

- **尾插法不维护尾指针：** 每次从头找尾会把建表复杂度提高到 O(n^2)。
- **头结点 next 未初始化：** 头插法依赖 L->next，若其中有脏数据会导致链表尾部异常。
- **忘记头插法会逆序：** 每个新结点都插到最前面，最终顺序与输入顺序相反。

## 本节总结

- 尾插法保持输入顺序，使用尾指针可做到 O(n)。
- 头插法每次插到表头，最终链表是输入序列的逆序。
- 头插法思想常用于链表逆置题。
