1.0 开篇：数据结构在学什么

把现实世界的问题信息化，并用计算机高效处理

打开单课页面

学习目标

  • 理解数据结构课程关注的两个核心问题：表示信息与高效处理信息。
  • 能把现实场景抽象为变量、数组、队列、图等数据组织形式。
  • 知道数据结构与 C 语言、算法、计算机组成、操作系统、计算机网络之间的关系。

知识点目录

  • 从现实世界到计算机中的数据：数据结构这门课首先要解决的是：怎样用程序代码把现实世界中的问题信息化。金钱可以被表示为数值，排队可以被表示为一组有先后关系的数据，社交网络中的关注关系则需要表达...
  • 信息化之后还要追求高效处理：把现实问题信息化只是第一步。真正产生价值的是让计算机高效处理这些信息，例如排队叫号能节省顾客等待成本，社交网络能快速维护和查询用户关系。
  • 与计算机专业基础课的关系：C 语言提供了变量、数组、结构体、指针、动态内存等表达工具；数据结构则教我们如何把这些工具组织成能解决问题的数据模型。

从现实世界到计算机中的数据

数据结构这门课首先要解决的是：怎样用程序代码把现实世界中的问题信息化。金钱可以被表示为数值，排队可以被表示为一组有先后关系的数据，社交网络中的关注关系则需要表达用户之间更复杂的连接关系。

信息化不是单纯把东西录入电脑，而是要选出合适的数据元素、数据项以及它们之间的关系。只有把现实对象变成计算机能识别和处理的数据，后续的查找、排序、插入、删除、推荐等操作才有基础。

关键提示
  • 现实对象进入程序后，需要变成数据、关系和操作。
  • 不同场景需要不同的数据组织方式。
本部分重点：数据结构的第一层意义是建模：把现实对象抽象成数据元素，把对象之间的联系抽象成结构。

信息化之后还要追求高效处理

把现实问题信息化只是第一步。真正产生价值的是让计算机高效处理这些信息，例如排队叫号能节省顾客等待成本，社交网络能快速维护和查询用户关系。

同样是表示排队，用数组、链表、队列等方式实现，插入、删除、查找的效率会不同。数据结构课程会不断比较这些方案，让我们知道在什么场景下选择哪种结构更合适。

关键提示
  • 好的数据组织方式能降低时间和空间开销。
本部分重点：数据结构不仅关心能不能表示问题，还关心处理这些数据是否高效。

与计算机专业基础课的关系

C 语言提供了变量、数组、结构体、指针、动态内存等表达工具；数据结构则教我们如何把这些工具组织成能解决问题的数据模型。

计算机组成原理帮助理解 CPU、内存等硬件如何工作；操作系统帮助理解系统如何管理资源；计算机网络解释不同设备之间如何传递信息。数据结构位于应用开发的核心位置，因为多数应用都在组织并处理数据。

关键提示
  • 数据结构连接了编程语言和现实应用。
本部分重点：C 语言是表达工具，数据结构是组织数据的思路，算法是在结构之上处理数据的方法。

代码整理

用结构体表示排队取号信息

原课用排队系统说明信息化。这里把一波顾客抽象为一个数据元素，字段就是数据项。

用结构体表示排队取号信息C
#include <stdio.h>

typedef struct {
    int ticketNo;
    int people;
    int takeTimeMinute;
} CustomerTicket;

int main(void) {
    CustomerTicket t = { 18, 4, 12 };
    printf("号数:%d 人数:%d 取号时间:%d分钟\n",
           t.ticketNo, t.people, t.takeTimeMinute);
    return 0;
}

易错点

把数据结构理解成只会写代码：数据结构更重要的是抽象现实关系，代码只是实现这种抽象的工具。
只关注能否存储，不关注效率：同一批数据可以有多种存储方式，后续操作的时间和空间代价可能相差很大。
忽视 C 语言基础：数组、结构体、指针和 malloc/free 是学习后续顺序表、链表的基础。

本节总结

  • 数据结构研究如何表示现实信息，以及如何高效处理这些信息。
  • 变量、数组、结构体、指针等 C 语言工具会在后续课程中被组合成各种数据结构。
  • 理解数据结构能帮助我们更好地理解信息化应用背后的数据组织方式。

自测题

1. 数据结构课程首先关注的核心问题是什么？
答案：B。课程开篇强调的是现实世界问题的信息化，以及后续高效处理。
2. 排队叫号系统中的一波顾客更适合作为什么？
答案：A。一波顾客包含号数、人数、取号时间等数据项，整体可作为一个数据元素。
3. 为什么仅用一个 float 变量不能表达微博关注关系？
答案：A。关注关系涉及用户之间的连接，属于关系结构，不是一个简单数值。

1.1 数据结构的基本概念

数据、数据元素、逻辑结构、存储结构与抽象数据类型

打开单课页面

学习目标

  • 区分数据、数据对象、数据元素、数据项和组合项。
  • 掌握数据结构的三要素：逻辑结构、存储结构、数据运算。
  • 理解抽象数据类型只定义逻辑结构和运算，不规定具体存储实现。

知识点目录

  • 数据、数据元素与数据项：数据是信息的载体，是能输入计算机并被程序识别、处理的符号集合。从计算机角度看，现实世界中的文字、数字、图片、关系最终都需要被转换成二进制形式。
  • 数据结构与数据对象：数据结构是相互之间存在一种或多种特定关系的数据元素的集合，强调数据元素之间的关系。数据对象是具有相同性质的数据元素的集合，不一定强调元素之间是否有关系。
  • 数据结构的三要素：逻辑结构描述数据元素之间的逻辑关系，常见类型有集合、线性结构、树形结构、图状或网状结构。线性结构是一对一关系，树形结构是一对多关系，图状结构是多对多关系。
  • 数据类型与抽象数据类型：数据类型是一个值的集合和定义在该集合上的一组操作。原子类型的值不可再分，如 bool、int；结构类型的值可以拆成多个分量，如 C 语言结构体。

数据、数据元素与数据项

数据是信息的载体，是能输入计算机并被程序识别、处理的符号集合。从计算机角度看，现实世界中的文字、数字、图片、关系最终都需要被转换成二进制形式。

数据元素是数据的基本单位，通常作为整体处理；数据项是组成数据元素的更小字段。以排队系统为例，一波顾客是一个数据元素，号数、取号时间、就餐人数是数据项。生日这种信息还可以拆成年、月、日等更小数据项，多个数据项组合成组合项。

关键提示
  • 数据元素是整体，数据项是组成整体的字段。
本部分重点：数据元素与数据项不是死记概念，而是由业务需求决定的抽象层次。

数据结构与数据对象

数据结构是相互之间存在一种或多种特定关系的数据元素的集合，强调数据元素之间的关系。数据对象是具有相同性质的数据元素的集合，不一定强调元素之间是否有关系。

例如某门店排队顾客之间有明确先后顺序，可以构成一个数据结构；所有门店的排队顾客信息具有相同性质，可以看作同一个数据对象。

关键提示
  • 结构的核心是关系。
本部分重点：数据结构强调元素之间的关系，数据对象强调元素具有相同性质。

数据结构的三要素

逻辑结构描述数据元素之间的逻辑关系，常见类型有集合、线性结构、树形结构、图状或网状结构。线性结构是一对一关系，树形结构是一对多关系，图状结构是多对多关系。

存储结构也叫物理结构，描述逻辑关系在计算机中的实现方式，常见有顺序存储、链式存储、索引存储和散列存储。顺序存储要求逻辑相邻的元素物理上也相邻；链式存储用指针表示后继关系；索引存储额外建立索引表；散列存储根据关键字计算地址。

数据运算是针对逻辑结构定义的操作，如队列的入队、出队；具体实现则依赖存储结构。同样是插入队尾，顺序存储和链式存储的实现细节完全不同。

关键提示
  • 逻辑结构面向问题，存储结构面向计算机，运算连接需求和实现。
本部分重点：学习任何具体数据结构，都要同时问：逻辑关系是什么、如何存储、要支持哪些运算。

数据类型与抽象数据类型

数据类型是一个值的集合和定义在该集合上的一组操作。原子类型的值不可再分，如 bool、int；结构类型的值可以拆成多个分量，如 C 语言结构体。

抽象数据类型 ADT 是对数据组织及相关操作的抽象定义。它关心逻辑结构和运算，不关心底层到底用顺序存储、链式存储还是其他存储方式实现。

关键提示
  • 抽象数据类型不绑定具体存储结构。
本部分重点：ADT 把“是什么”和“能做什么”先定义清楚，把“怎么存、怎么写代码”留给实现阶段。

代码整理

数据元素与数据项示例

用结构体表达排队顾客信息，展示数据元素由多个数据项组成。

数据元素与数据项示例C
typedef struct {
    int ticketNo;      // 关键字：排队号，可区分不同顾客
    int people;        // 数据项：就餐人数
    int takeHour;      // 数据项：取号小时
    int takeMinute;    // 数据项：取号分钟
} CustomerInfo;

易错点

混淆数据对象和数据结构：数据对象强调同性质，数据结构强调元素之间有特定关系。
把逻辑结构等同于存储结构：线性表是逻辑结构，顺序表和链表是不同存储实现。
认为 ADT 必须说明 malloc 或数组：ADT 只定义逻辑组织和操作，不规定底层存储。

本节总结

  • 数据结构由逻辑结构、存储结构和数据运算三要素共同描述。
  • 常见逻辑结构包括集合、线性结构、树形结构、图状结构。
  • 常见存储结构包括顺序、链式、索引和散列存储。
  • 抽象数据类型先描述逻辑和操作，再由具体代码选择存储方式实现。

自测题

1. 数据结构最强调什么？
答案：B。结构本质上就是元素之间的关系。
2. 下列哪一项属于逻辑结构？
答案：C。树形结构描述元素之间一对多的逻辑关系。
3. 抽象数据类型 ADT 不关心哪件事？
答案：C。ADT 是抽象定义，具体存储属于实现阶段。

1.2.1 算法的基本概念

算法定义、五个特性和好算法的标准

打开单课页面

学习目标

  • 理解算法是求解特定问题的有限步骤描述。
  • 掌握算法必须具备的有穷性、确定性、可行性、输入、输出。
  • 理解好算法还应具备正确性、可读性、健壮性、高效率和低存储需求。

知识点目录

  • 程序、数据结构与算法：经典说法“程序 = 数据结构 + 算法”强调了两件事：数据结构负责组织和表示问题中的信息，算法负责按明确步骤处理这些数据，从而解决实际问题。
  • 算法的五个必要特性：有穷性要求算法在有限步内结束，每一步也能在有限时间完成。一直运行的应用程序或死循环不是算法。
  • 好算法的评价标准：正确性是第一要求：算法必须能正确解决问题。一个能运行但排序结果错误的过程仍可能是算法，却不是好算法。

程序、数据结构与算法

经典说法“程序 = 数据结构 + 算法”强调了两件事：数据结构负责组织和表示问题中的信息，算法负责按明确步骤处理这些数据，从而解决实际问题。

做番茄炒蛋需要食材和步骤；程序也类似，数据结构像食材，算法像加工步骤。没有合适的数据，算法无从处理；没有算法，数据也不能产生价值。

关键提示
  • 算法是处理数据的步骤描述。
本部分重点：数据结构解决“处理什么”，算法解决“怎样处理”。

算法的五个必要特性

有穷性要求算法在有限步内结束，每一步也能在有限时间完成。一直运行的应用程序或死循环不是算法。

确定性要求每条指令含义明确，对相同输入必须得到相同输出。若排序时遇到年龄相同的人没有规定稳定规则，结果可能前后不一致，就不满足确定性。

可行性要求每个步骤能被计算机实现。算法可以有零个或多个输入，但必须有一个或多个输出，且输出与输入之间有确定关系。

关键提示
  • 算法的描述必须明确、可执行、会结束。
本部分重点：只要缺少有穷性、确定性、可行性、输入、输出中的任意一项，就不能称为严格意义上的算法。

好算法的评价标准

正确性是第一要求：算法必须能正确解决问题。一个能运行但排序结果错误的过程仍可能是算法，却不是好算法。

可读性要求算法描述或代码容易理解，便于协作和维护；健壮性要求面对非法输入时能作出合理反应，而不是产生莫名其妙的结果。

高效率和低存储需求分别对应后续要学习的时间复杂度和空间复杂度。实际开发中，问题规模一大，低效算法会暴露明显性能问题。

关键提示
  • 正确性是底线，效率和健壮性决定工程质量。
本部分重点：好算法不仅要能算对，还要容易读、能处理边界、时间和空间开销可接受。

代码整理

按年龄递增排序并处理同龄情况

原课用按年龄排序说明算法步骤和确定性。这里给出稳定选择排序的 C 实现：年龄相同则保持原先相对顺序。

按年龄递增排序并处理同龄情况C
#include <stdio.h>

typedef struct {
    char name[16];
    int age;
} Person;

void stableSelectByAge(Person a[], int n) {
    for (int i = 0; i < n - 1; ++i) {
        int min = i;
        for (int j = i + 1; j < n; ++j) {
            if (a[j].age < a[min].age) {
                min = j;
            }
        }
        Person temp = a[min];
        for (int k = min; k > i; --k) {
            a[k] = a[k - 1];
        }
        a[i] = temp;
    }
}

易错点

把程序都叫算法：程序可以长期运行，算法必须有穷。微信是程序，不是一个单独算法。
忽略确定性：相同输入必须有相同输出，模糊的步骤描述会破坏确定性。
只看结果不看健壮性：好算法要处理非法输入，避免边界条件造成错误。

本节总结

  • 算法是对特定问题求解步骤的有限描述。
  • 算法必须具有有穷性、确定性、可行性、输入和输出。
  • 好算法要正确、可读、健壮，并尽量高效、节省空间。

自测题

1. 死循环为什么不能称为算法？
答案：B。算法必须在有限步内结束。
2. “找到最小年龄的人”在有两人同龄时还需要补充规则，主要是为了满足什么？
答案：B。相同输入应得到相同输出，因此要规定同龄时如何处理。
3. 高效率和低存储需求分别对应后续哪两个概念？
答案：B。效率用时间复杂度衡量，存储开销用空间复杂度衡量。

1.2.2 算法的时间复杂度

用大 O 表示法分析算法运行时间随问题规模的增长

打开单课页面

学习目标

  • 理解为什么不能只用事后统计运行时间评价算法。
  • 掌握大 O 表示法、加法规则、乘法规则和常见数量级顺序。
  • 能分析循环、嵌套循环、对数循环和查找算法的最好、最坏、平均时间复杂度。

知识点目录

  • 为什么需要时间复杂度：直接把程序跑一遍统计时间，会受到机器性能、编程语言、编译器质量和测试环境影响，不能客观反映算法本身优劣。有些算法甚至不能等运行后再统计，例如高风险控制系统。
  • 大 O 表示法与数量级：分析时间复杂度时，只保留表达式中增长最快的最高阶项，并把常数系数化为 1。例如 3n + 3 记为 O(n)，n^2 + 6n + 100 记为 O(n^2)。
  • 循环、嵌套循环与对数循环：顺序执行的语句只影响常数项，通常可以忽略。含循环的程序重点看循环体中的基本操作执行了多少次；多层嵌套循环重点看最深层循环总共执行多少次。
  • 最好、最坏和平均时间复杂度：有些算法的运行时间和输入数据状态有关。在线性查找中，如果目标元素在第一个位置，最好时间复杂度是 O(1)；如果在最后一个位置，最坏时间复杂度是 O(n)。

为什么需要时间复杂度

直接把程序跑一遍统计时间，会受到机器性能、编程语言、编译器质量和测试环境影响，不能客观反映算法本身优劣。有些算法甚至不能等运行后再统计，例如高风险控制系统。

时间复杂度关注的是算法执行时间 T 与问题规模 n 之间的增长关系。问题规模越大，算法运行时间通常越长，我们希望在运行前就估计这种增长趋势。

关键提示
  • 评价算法要排除与算法设计无关的外界因素。
本部分重点：时间复杂度不是精确秒数，而是问题规模扩大时运行时间的增长数量级。

大 O 表示法与数量级

分析时间复杂度时，只保留表达式中增长最快的最高阶项，并把常数系数化为 1。例如 3n + 3 记为 O(n)，n^2 + 6n + 100 记为 O(n^2)。

常见增长顺序可以记为：常数阶 O(1) < 对数阶 O(log n) < 线性阶 O(n) < 线性对数阶 O(n log n) < 平方阶 O(n^2) < 立方阶 O(n^3) < 指数阶 O(2^n) < 阶乘阶 O(n!)。

多项相加保留数量级最高的一项，这是加法规则；循环嵌套时数量级相乘，这是乘法规则。

关键提示
  • 先找基本操作，再看它执行次数与 n 的关系。
本部分重点：大 O 表示法抓主导项：低阶项和常数系数在大规模问题下可以忽略。

循环、嵌套循环与对数循环

顺序执行的语句只影响常数项，通常可以忽略。含循环的程序重点看循环体中的基本操作执行了多少次；多层嵌套循环重点看最深层循环总共执行多少次。

如果循环变量每次加 1，通常是线性阶 O(n)；两层都与 n 相关的嵌套循环通常是 O(n^2)；如果循环变量每次翻倍，循环次数大约是 log2 n，因此是 O(log n)。

关键提示
  • 最深层循环的执行次数往往决定复杂度。
本部分重点：循环复杂度的核心问题是：基本操作到底执行了多少次。

最好、最坏和平均时间复杂度

有些算法的运行时间和输入数据状态有关。在线性查找中，如果目标元素在第一个位置，最好时间复杂度是 O(1)；如果在最后一个位置，最坏时间复杂度是 O(n)。

平均时间复杂度通常假设目标出现在各位置的概率相同，再计算期望执行次数。线性查找平均需要约 (n+1)/2 次比较，因此仍为 O(n)。实际评价算法时更常关注最坏和平均情况，最好情况参考价值较小。

关键提示
  • 输入分布会影响执行次数。
本部分重点：同一个算法可能有不同情况下的复杂度，考试和工程中通常重点看最坏与平均情况。

代码整理

三个典型时间复杂度示例

对应课程中的逐步递增、翻倍循环和线性查找。

三个典型时间复杂度示例C
#include <stdio.h>

void loveLinear(int n) {
    for (int i = 1; i <= n; ++i) {
        printf("%d\n", i);       // O(n)
    }
}

void loveLog(int n) {
    for (int i = 1; i <= n; i *= 2) {
        printf("%d\n", i);       // O(log n)
    }
}

int locateN(const int a[], int n) {
    for (int i = 0; i < n; ++i) {
        if (a[i] == n) {
            return i;            // 最好 O(1)，最坏 O(n)
        }
    }
    return -1;
}

易错点

把运行秒数当成时间复杂度：秒数受机器、语言和编译器影响，复杂度关心增长数量级。
没有去掉低阶项和常数系数：3n+3、100n+8 都是 O(n)，不能把常数误当成复杂度核心。
忽视输入状态：线性查找的最好、最坏、平均情况可能不同，要按题意分析。

本节总结

  • 时间复杂度描述 T(n) 的增长数量级，而不是精确运行时间。
  • 大 O 记法保留最高阶项并去掉常数系数。
  • 循环分析重点是基本操作执行次数，嵌套循环常用乘法规则。
  • 线性查找等算法需要区分最好、最坏和平均情况。

自测题

1. 3n + 100 的时间复杂度是？
答案：C。大 O 记法去掉低阶项和常数系数。
2. 循环 `for (i=1; i<=n; i*=2)` 的复杂度通常是？
答案：B。每次翻倍，执行次数约为 log2 n。
3. 在线性查找中，目标在最后一个位置时的时间复杂度是？
答案：C。需要从头扫描到最后一个元素。

程序运行时的内存开销

程序运行前，代码会被加载到内存中；运行时还需要保存参数、局部变量、临时数据等。代码区大小通常和问题规模 n 无关，空间复杂度重点关注会随 n 变化的额外空间。

如果一个算法只使用固定数量的变量，即使 n 变大，所需额外空间也不变，它的空间复杂度是 O(1)，这种算法也称为可以原地工作。

关键提示
  • 固定数量变量通常只贡献常数空间。
本部分重点：空间复杂度关注随问题规模变化的额外内存，而不是所有内存细节。

数组变量带来的空间复杂度

若函数中定义了长度为 n 的数组，数组空间随 n 线性增长，空间复杂度是 O(n)。若定义 n x n 的二维数组，空间开销随 n^2 增长，空间复杂度是 O(n^2)。

分析时不必纠结 int 到底占几个字节，因为大 O 表示法最终会去掉常数系数。我们只需判断哪些变量的大小与问题规模相关，并保留最高阶项。

关键提示
  • 数组长度直接影响空间数量级。
本部分重点：普通程序的空间复杂度通常看与 n 相关的数据结构大小。

递归调用栈与空间复杂度

函数每调用一次，系统都要为该层调用保存参数、局部变量和返回地址等信息，这些信息会形成函数调用栈。递归调用会产生多层栈帧，因此递归深度会影响空间复杂度。

若每层递归只使用常数空间，且递归深度为 n，则空间复杂度为 O(n)。若每层还定义长度与当前参数相关的数组，总空间可能变成 1 + 2 + ... + n，即 O(n^2)。

关键提示
  • 递归不是免费调用，每一层都会占用调用栈空间。
本部分重点：大多数递归题的空间复杂度可以先看递归深度，再考虑每层是否有额外大数组。

代码整理

空间复杂度示例

展示 O(1)、O(n) 以及递归 O(n) 的典型来源。

空间复杂度示例C
void constantSpace(int n) {
    int i = 0;
    int sum = 0;       // 变量个数固定，S(n)=O(1)
    while (i <= n) {
        sum += i;
        i++;
    }
}

void linearSpace(int n) {
    int a[n];          // 数组长度随 n 变化，S(n)=O(n)
    for (int i = 0; i < n; ++i) {
        a[i] = i;
    }
}

void recursivePrint(int n) {
    if (n <= 0) {
        return;
    }
    recursivePrint(n - 1);  // 递归深度 n，调用栈 S(n)=O(n)
}

易错点

把所有内存都算进空间复杂度：空间复杂度通常关注随 n 增长的额外空间，代码区和固定变量多为常数项。
递归只看代码里有没有数组：即使没有数组，递归调用栈也会占空间。
过度纠结具体字节数：大 O 分析关注数量级，int 是 4 字节还是其他值通常不影响结果。

本节总结

  • 空间复杂度 S(n) 描述算法额外空间随问题规模的增长关系。
  • 固定变量是 O(1)，长度为 n 的数组是 O(n)，n x n 数组是 O(n^2)。
  • 递归空间通常由递归深度和每层调用的空间共同决定。

自测题

1. 只使用固定几个 int 变量的算法空间复杂度通常是？
答案：A。固定数量变量不随 n 增长。
2. 定义 `int a[n][n]` 的主要空间数量级是？
答案：D。二维数组有 n*n 个元素。
3. 递归深度为 n，且每层只用常数空间时，总空间复杂度是？
答案：B。每层常数空间乘以 n 层调用栈。
