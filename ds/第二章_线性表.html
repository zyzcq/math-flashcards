1. 线性表的定义和基本操作

线性表是具有相同数据类型的 n 个数据元素的有限序列。若 n=0，就是空表。通常写作：

记忆式：L = (a1, a2, ..., ai, ..., an)。除了第一个元素没有直接前驱、最后一个元素没有直接后继，其余元素都有且仅有一个直接前驱和一个直接后继。

线性表的逻辑结构：一对一，像排队。只谈“谁在谁前后”，先别管它们住在内存哪里。

线性表常见基本操作

操作含义考点提醒
InitList(&L)初始化表，构造空表。带 & 表示会修改 L 本身。
DestroyList(&L)销毁线性表并释放资源。动态分配空间时尤其要注意。
ListInsert(&L,i,e)在第 i 个位置插入元素 e。合法范围通常是 1 到 length+1。
ListDelete(&L,i,&e)删除第 i 个位置元素，并用 e 返回。合法范围是 1 到 length。
LocateElem(L,e)按值查找元素 e。通常要顺序扫描。
GetElem(L,i)按位序查找第 i 个元素。顺序表 O(1)，链表 O(n)。
Length(L)、Empty(L)、PrintList(L)求表长、判空、输出。属于基础操作，不同存储结构实现不同。
为什么有的参数要写 &L 或 &e？

一句话：函数里改了它，还要把修改结果带回调用者，就要传引用。初始化、插入、删除会改变表本身，所以常见 &L；删除时若要把被删元素带回来，就写 &e。单纯查找、求长度、判空不需要改原表，通常传 L 就够了。这里是 C++ 引用写法；如果用纯 C，常改成指针参数来完成同样的事。

封装这些基本操作，不是为了显得代码很有仪式感，而是为了团队协作和避免重复劳动：以后想增删改查，调用函数就行，别每次都从零手搓一遍。

2. 顺序表：用连续空间实现线性表

顺序表是线性表的顺序存储实现。它用一整片连续的内存空间，依次存放线性表中的元素。

顺序表像电影院连号座：知道第一个座位，就能算出第 i 个座位在哪。

静态分配

typedef struct {
    ElemType data[MaxSize];
    int length;
} SqList;

静态分配的数组容量固定。优点是定义简单；缺点是空间一开始就定死，表满后不能直接扩容。

动态分配

#define InitSize 10
typedef struct {
    ElemType *data;
    int MaxSize;
    int length;
} SeqList;

void InitList(SeqList &L) {
    L.data = (ElemType *)malloc(InitSize * sizeof(ElemType));
    L.length = 0;
    L.MaxSize = InitSize;
}

动态分配并不是“无限长”。它只是运行时申请一片连续空间；想扩容时，通常需要申请更大的新空间、复制旧元素、释放旧空间。

实现方式容量特点跨考提醒
静态分配容量编译时确定，数组大小固定。表满后不能继续插入，除非重新定义更大的结构。
动态分配运行时用 malloc 申请连续空间，可扩容。扩容不是原地变长，通常要申请新空间、复制、释放旧空间。
易错点：位序从 1 开始，下标从 0 开始。第 i 个元素对应数组下标 i-1。
主动回忆：顺序表三件套
请闭眼说出顺序表为什么支持随机存取：连续存储、元素等长、已知起始地址，所以能用地址公式直接算第 i 个元素。

3. 顺序表插入、删除、查找

插入：先搬家，再入住

在第 i 个位置插入 e，合法范围是 1 <= i <= L.length + 1。如果表满，插入失败。

bool ListInsert(SqList &L, int i, ElemType e) {
    if (i < 1 || i > L.length + 1) return false;
    if (L.length >= MaxSize) return false;
    for (int j = L.length; j >= i; j--)
        L.data[j] = L.data[j - 1];
    L.data[i - 1] = e;
    L.length++;
    return true;
}

在第 2 个位置插入 X：从后往前搬，给 X 腾座位。

删除：先取走，再补位

删除第 i 个位置元素，合法范围是 1 <= i <= L.length。删除后，后面的元素依次前移。

bool ListDelete(SqList &L, int i, ElemType &e) {
    if (i < 1 || i > L.length) return false;
    e = L.data[i - 1];
    for (int j = i; j < L.length; j++)
        L.data[j - 1] = L.data[j];
    L.length--;
    return true;
}

删除的最好情况是删表尾 O(1)，最坏情况是删表头 O(n)，平均移动约 (n-1)/2 个元素，数量级仍是 O(n)。

查找：按位快，按值慢

查找方式顺序表表现原因
按位查找 GetElem(L,i)O(1)连续存储 + 元素等长，可直接套地址公式。
按值查找 LocateElem(L,e)O(n)不知道 e 在哪，只能从前往后比。
操作合法范围移动方向复杂度来源
插入第 i 个位置1 <= i <= length + 1从后往前移，避免覆盖。插入位置后面的元素都要后移。
删除第 i 个位置1 <= i <= length从前往后补位。删除位置后面的元素都要前移。
按位查找1 <= i <= length不移动。地址公式直接定位。
主动回忆：顺序表插删为什么是 O(n)
不要只背结论。请分别说明：插入表头为什么要移动 n 个元素，删除表头为什么要移动 n-1 个元素，为什么平均情况仍然记为 O(n)。

4. 单链表：元素散坐，指针认路

单链表使用链式存储。每个结点包含数据域和指针域，指针域 next 指向后继结点。

单链表不要求物理相邻，只要求用 next 把逻辑顺序串起来。

typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;
头指针指向链表第一个结点。带头结点时，它指向头结点；不带头结点时，它指向第一个数据结点。
头结点不一定存数据，主要让插入删除更统一。带头结点的空表满足 L->next == NULL。
首元结点第一个真正存放数据元素的结点。别把头结点和首元结点认成一个人。
bool InitList(LinkList &L) {
    L = (LNode *)malloc(sizeof(LNode));
    if (L == NULL) return false;
    L->next = NULL;
    return true;
}
实现方式空表条件代码处理差异
带头结点L != NULL 且 L->next == NULL第 1 个位置的插入删除可统一看成在头结点后操作。
不带头结点L == NULL操作第 1 个数据结点时常要单独修改头指针。
跨考提醒：LinkList 本质上是 LNode *，但名字强调“这是一条链表”；LNode *p 强调“这是一个结点指针”。类型相同，语义不同。

5. 单链表插入、删除、查找、建立

按位插入：先找前驱，再改指针

在第 i 个位置插入元素，需要先找到第 i-1 个结点 p，再把新结点 s 接到 p 后面。

bool InsertNextNode(LNode *p, ElemType e) {
    if (p == NULL) return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) return false;
    s->data = e;
    s->next = p->next;  // 先让新人认后门
    p->next = s;        // 再让前驱认新人
    return true;
}
指针顺序别写反：先 s->next = p->next，再 p->next = s。反过来会把后半截链表弄丢，像搬家时先把旧地址撕了。

后插操作：新结点先接后继，再被前驱接纳。

前插操作：可找前驱，也可后插后交换数据

如果只给定结点 p，想在 p 前插入 s，单链表不能直接找到 p 的前驱。常见技巧是：把 s 插到 p 后面，然后交换 p 和 s 的数据域，效果等价于在 p 前插入。

删除结点

bool DeleteNextNode(LNode *p) {
    if (p == NULL || p->next == NULL) return false;
    LNode *q = p->next;
    p->next = q->next;
    free(q);
    return true;
}

如果给定结点 p 本身并要求删除 p，可用“复制后继数据再删后继”的技巧：令 q=p->next，把 q 的数据复制到 p，再删除 q。注意：如果 p 是最后一个结点，这招不成立。

查找

操作单链表复杂度原因
按位查找第 i 个结点O(n)只能从头结点一路数过去。
按值查找 eO(n)只能逐个比较数据域。
已知前驱 p 后插/删后继O(1)只改几个指针，不搬元素。

建立单链表：尾插法与头插法

尾插法 维护尾指针 r，每次把新结点接到尾部，输入顺序和链表顺序一致。适合“按原顺序建表”。
头插法 每次把新结点插到头结点后面，最终顺序与输入顺序相反。适合逆置链表，像每来一个人都插队到最前面。
操作必须先做什么最容易错哪儿
后插结点检查 p 和 malloc 结果。先改 p->next 导致原后继丢失。
删除后继保存 q = p->next。忘记 free(q)，或 p->next == NULL 时仍删除。
按位插入先找到第 i-1 个结点。把“第 i 个结点”和“第 i-1 个前驱”混淆。
建表明确要保序还是逆序。把头插法最终顺序误认为和输入顺序一致。
主动回忆：单链表四句口诀
按位插入先找前驱；后插先接后继再接前驱；删除先保存被删结点再断链释放；头插逆序、尾插保序。

6. 双链表：向前向后都认路

单链表给定某个结点 p 时，找后继很容易，找前驱很麻烦。双链表在每个结点中增加 prior 指针，指向前驱结点。

typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next;
} DNode, *DLinkList;

双链表每个结点有两根线：prior 向前，next 向后。

bool InsertNextDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL) return false;
    s->next = p->next;
    if (p->next != NULL)
        p->next->prior = s;
    s->prior = p;
    p->next = s;
    return true;
}
双链表插入要照顾四条线。若 p 是尾结点，p->next 为 NULL，不能直接写 p->next->prior。

删除结点时，要让前驱的 next 跳过它、后继的 prior 跳过它，再释放结点。双链表在“找前驱”这件事上比单链表舒服很多。

主动回忆：双链表为什么更方便
给定某个结点 p，单链表找前驱通常要从头遍历；双链表可以通过 p->prior 直接找到前驱。请顺着这句话解释双链表为什么会多一个指针域空间开销。

7. 循环链表：尾巴不指 NULL，转回头

循环链表是在普通链表基础上的小改造。循环单链表中，尾结点的 next 指回头结点；循环双链表中，头结点的 prior 也指向尾结点。

循环单链表：尾结点 next 指向头结点，形成闭环。

类型初始化/判空表尾判断特点
循环单链表L->next = L，空表 L->next == Lp->next == L给任意结点可继续绕回整表，但找前驱仍要绕。
循环双链表L->next=L; L->prior=Lp->next == L前后都能走，头尾衔接更顺。
主动回忆：循环链表不看 NULL
普通链表常用 NULL 判断结尾；循环链表的尾巴会指回头结点。请写出带头结点循环单链表的空表条件和表尾条件。

8. 静态链表：用数组模拟指针

静态链表用一整片连续数组来存放结点，但结点之间的逻辑关系不靠真实指针，而靠游标 next 保存下一个结点的数组下标。

#define MaxSize 1000
typedef struct {
    ElemType data;
    int next;
} SLinkList[MaxSize];

静态链表：数组下标充当“伪指针”。0 号单元常当头结点，-1 表示结束。

静态链表兼具“数组空间固定”和“链式逻辑关系”的味道。它不需要真实指针，适合没有指针的语言或需要在数组中模拟链表的场景；缺点是容量仍固定，且插入删除要维护空闲结点。

主动回忆：静态链表到底静在哪
静态链表的数组容量固定，所以“静”；它又用 next 游标保存下一个结点下标，所以逻辑上像链表。请说明 next=-1 和单链表 next=NULL 的对应关系。

9. 顺序表和链表的比较

顺序表和链表在逻辑上都属于线性表，区别主要来自存储结构。顺序表把元素连续摆放，链表让元素分散存放、靠指针相连。

角度顺序表链表
逻辑结构线性结构，一对一。线性结构，一对一。
存储结构连续空间，元素物理相邻。离散空间，指针维护逻辑关系。
随机存取支持，按位查找 O(1)。不支持，按位查找 O(n)。
按值查找通常 O(n)。通常 O(n)。
插入删除平均 O(n)，主要花在移动元素。找前驱 O(n)，已知前驱后改指针 O(1)。
空间存储密度高，但可能预分配浪费或溢出。动态灵活，但每个结点有指针域开销。
容量扩容不方便，需要新空间并复制。按需申请结点，容量更灵活。
适用场景表长稳定、频繁按位访问、插删少。表长变化大、插删频繁、随机访问少。
选择口诀：查得多、位置明确、规模稳定，顺序表很香；插删多、规模飘忽、空间不想一口气预留，链表更灵活。
简答题答题角度该写什么
逻辑结构二者都是线性表，元素之间都是一对一关系。
存储结构顺序表连续存储，链表离散存储并用指针表示逻辑关系。
基本操作顺序表按位查找快，链表插删改指针方便；具体复杂度要说明代价来源。
应用场景表长稳定且查找多选顺序表；长度变化大且插删多选链表。
主动回忆：顺序表和链表比较模板
请按“逻辑结构、存储结构、基本操作、应用场景”四个角度口述一遍。不要只说谁快，要说快在哪里、慢在哪里、为什么。

10. 复习区：线性表闭环自测

线性表定义同类型、有限序列、一对一关系；第一个无前驱，最后一个无后继。
顺序表三件事连续空间、长度 length、最大容量 MaxSize；第 i 个元素下标 i-1。
顺序表插入范围 1 到 length+1；从后往前移动；平均 O(n)。
顺序表删除范围 1 到 length；从前往后补位；平均 O(n)。
单链表三名词头指针、头结点、首元结点必须区分。
单链表插入先 s->next=p->next，再 p->next=s。
单链表删除删后继要 q=p->next，接链，再 free(q)。
头插尾插头插逆序，尾插保序。
双链表多一个 prior，找前驱 O(1)，插删要维护四条线。
循环链表空表和表尾判断不再看 NULL，而看是否回到头结点。
静态链表数组 + 游标 next，游标保存下一个结点下标，-1 表示结束。
最终比较顺序表随机访问强，链表插删灵活；没有谁永远赢，只有场景合不合适。
自测 1：为什么顺序表按位查找是 O(1)，链表是 O(n)？
顺序表连续存储且元素等长，可通过地址公式直接计算第 i 个元素位置；链表只能从头沿 next 一个个走到第 i 个。
自测 2：链表已知前驱插入为什么是 O(1)？
因为不用移动元素，只需申请新结点并修改两条指针：新结点指向原后继，前驱指向新结点。
自测 3：头插法和尾插法最终顺序有什么区别？
头插法每次插到最前面，最终顺序与输入顺序相反；尾插法每次接到尾部，最终顺序与输入顺序一致。
自测 4：循环单链表空表条件是什么？
带头结点的循环单链表初始化为 L->next = L，所以空表条件是 L->next == L。
自测 5：静态链表里的 next 和单链表里的 next 有何区别？
单链表 next 存真实指针或地址；静态链表 next 存数组下标，也叫游标，用下标模拟指针。
