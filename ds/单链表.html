单链表：定义、插删、查找、建表

这份页面把前面几节单链表课整理成一个学习和复习系统。主线是：结点结构、头指针与头结点、插入删除的改链逻辑、查找遍历、头插法与尾插法建表。

学习地图

总主线：单链表用结点里的 next 指针表示先后关系。插入、删除、查找、建表，本质都是围绕 next 的移动和修改。

先理解结构

结点由 data 和 next 组成。头指针找到第一个入口，之后顺着 next 找完整条链。

再练改链

插入和删除都要先找前驱。后插是基础，按位插入、按位删除都能复用它的思想。

最后会遍历

按位查找、按值查找、求表长、尾插建表，都是让指针从前往后扫描。

复习口诀：找结点靠遍历，插入先连后面再连前面，删除先保存目标再跳过释放，头插得到逆序，尾插保持原序。
一、单链表的定义与初始化

1. 顺序表和链表的区别

顺序表

[a1][a2][a3][a4] 物理上连续存放

优点是支持随机存取，存储密度高。缺点是需要大片连续空间，扩容不方便。

单链表

[a1|next] -> [a2|next] -> [a3|NULL] 物理上可以分散存放

优点是扩容、插入、删除更灵活。缺点是不能随机存取，只能从前往后找。

2. 结点结构

┌────────┬────────┐ │ data │ next │ └────────┴────────┘ 数据域 指针域

data 存数据元素，next 存下一个结点的地址。每个结点只有一个后继指针，所以叫单链表。

typedef int ElemType;

typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;
typedef 要拆开理解：LNode 等价于 struct LNode；LinkList 等价于 struct LNode *。
写法 本质 强调点
LNode *p 结点指针 强调 p 指向一个结点
LinkList L 结点指针 强调 L 是一条链表的入口

3. 头指针、头结点、第一个数据结点

不带头结点

L -> [10| ] -> [20|NULL] 第一个数据结点 空表：L == NULL

L 直接指向第一个数据结点。

带头结点

L -> [头结点| ] -> [10| ] -> [20|NULL] 第一个数据结点 空表：L->next == NULL

L 指向头结点，头结点不存实际数据，L->next 才是第一个数据结点。

4. 初始化与判空

不带头结点

bool InitListNoHead(LinkList *L) {
    *L = NULL;
    return true;
}

bool EmptyNoHead(LinkList L) {
    return L == NULL;
}

带头结点

bool InitList(LinkList *L) {
    *L = (LNode *)malloc(sizeof(LNode));
    if (*L == NULL) {
        return false;
    }

    (*L)->next = NULL;
    return true;
}

bool Empty(LinkList L) {
    return L->next == NULL;
}
注意：教材里有时写 LinkList &L，这是 C++ 引用写法。标准 C 里要用 LinkList *L，因为要修改头指针本身。
本部分重点：单链表的关键是 data + next。带头结点写代码更统一，因为可以把头结点看成第 0 个结点；不带头结点处理第一个结点时常要特殊判断。
二、单链表的插入与删除

1. 带头结点按位插入

在第 i 个位置插入元素 e，要先找到第 i - 1 个结点，因为真正要改的是前驱结点的 next。

插入前： p -> 原第 i 个结点 插入后： p -> s -> 原第 i 个结点
bool ListInsert(LinkList L, int i, ElemType e) {
    if (i < 1) {
        return false;
    }

    LNode *p = L;
    int j = 0;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL) {
        return false;
    }

    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) {
        return false;
    }

    s->data = e;
    s->next = p->next;
    p->next = s;

    return true;
}
两句不能颠倒：必须先写 s->next = p->next，再写 p->next = s。如果反过来，s->next 可能指向自己，后面的链会断。

2. 不带头结点按位插入

不带头结点时没有第 0 个结点，所以插入第 1 个位置要单独处理。

bool ListInsertNoHead(LinkList *L, int i, ElemType e) {
    if (i < 1) {
        return false;
    }

    if (i == 1) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return false;
        }

        s->data = e;
        s->next = *L;
        *L = s;
        return true;
    }

    LNode *p = *L;
    int j = 1;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL) {
        return false;
    }

    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) {
        return false;
    }

    s->data = e;
    s->next = p->next;
    p->next = s;
    return true;
}

3. 后插操作

给定结点 p，在它后面插入新元素。这个操作是很多插入算法的基础。

bool InsertNextNode(LNode *p, ElemType e) {
    if (p == NULL) {
        return false;
    }

    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) {
        return false;
    }

    s->data = e;
    s->next = p->next;
    p->next = s;
    return true;
}
封装好处：按位插入可以先找第 i - 1 个结点，再调用 InsertNextNode(p, e)。代码更短，边界处理也更集中。

4. 前插操作

只给定结点 p，要在它前面插入新元素。单链表不能从 p 往前找，所以有两种思路。

思路一：从头找前驱

如果给了头指针 L，可以从表头遍历，找到 p 的前驱，再后插。时间复杂度 O(n)。

思路二：后插再换数据

先把新结点插到 p 后面，再交换 p 和新结点的数据。逻辑效果等价于前插。时间复杂度 O(1)。

原来： p(x) -> 后继 先后插： p(x) -> s(e) -> 后继 交换数据： p(e) -> s(x) -> 后继
bool InsertPriorNode(LNode *p, ElemType e) {
    if (p == NULL) {
        return false;
    }

    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL) {
        return false;
    }

    s->next = p->next;
    p->next = s;

    s->data = p->data;
    p->data = e;
    return true;
}

5. 按位删除

删除第 i 个结点，也要先找第 i - 1 个结点。

删除前： p -> q -> q的后继 删除后： p ------> q的后继 最后 free(q)
bool ListDelete(LinkList L, int i, ElemType *e) {
    if (i < 1) {
        return false;
    }

    LNode *p = L;
    int j = 0;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL || p->next == NULL) {
        return false;
    }

    LNode *q = p->next;
    *e = q->data;
    p->next = q->next;
    free(q);
    return true;
}

6. 删除给定结点

只给定 p，删除它。如果 p 不是尾结点，可以用后继覆盖法。

原来： p(x) -> q(y) -> 后继 复制后继数据： p(y) -> q(y) -> 后继 删除 q： p(y) -> 后继
bool DeleteNode(LNode *p) {
    if (p == NULL || p->next == NULL) {
        return false;
    }

    LNode *q = p->next;
    p->data = q->data;
    p->next = q->next;
    free(q);
    return true;
}
尾结点不能这样删：如果 p->next == NULL，q 就是空指针，再访问 q->data 会出错。删除尾结点只能从头开始找前驱，时间复杂度 O(n)。
本部分重点：插入核心是 s->next = p->next; p->next = s;。删除核心是保存目标结点 q，让前驱跳过它，再 free(q)。只给结点指针时，前插和删除都可以用“数据搬家”的技巧，但删除尾结点不行。
三、单链表的查找与求长度

1. 按位查找

带头结点时，可以把头结点看成第 0 个结点。按位查找就是让 p 往后走 i 次。

LNode *GetElem(LinkList L, int i) {
    if (i < 0) {
        return NULL;
    }

    LNode *p = L;
    int j = 0;

    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }

    return p;
}
边界：i == 0 返回头结点；i 超过表长时最终返回 NULL。调用者可以通过返回值是否为 NULL 判断是否查找成功。

2. 按值查找

按值查找从第一个数据结点开始，找到第一个 data == e 的结点就返回。

LNode *LocateElem(LinkList L, ElemType e) {
    LNode *p = L->next;

    while (p != NULL && p->data != e) {
        p = p->next;
    }

    return p;
}
结构体比较：如果 ElemType 是结构体，不能简单写 p->data == e。要自己写比较函数，按字段比较。

3. 求表长

int Length(LinkList L) {
    int len = 0;
    LNode *p = L->next;

    while (p != NULL) {
        len++;
        p = p->next;
    }

    return len;
}

4. 不带头结点的区别

区别只有入口不同：带头结点从 L->next 处理数据结点，不带头结点从 L 处理数据结点。

LNode *GetElemNoHead(LinkList L, int i) {
    if (i < 1) {
        return NULL;
    }

    LNode *p = L;
    int j = 1;

    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }

    return p;
}
本部分重点：查找类操作的核心都是 p = p->next。单链表不能随机访问，所以按位查找、按值查找、求表长平均和最坏都是 O(n)。
四、尾插法和头插法建立单链表

1. 尾插法

尾插法每次把新结点插到表尾，建立结果和输入顺序一致。

输入：10 16 27 结果：L -> 头结点 -> 10 -> 16 -> 27 -> NULL

如果每次都调用按位插入插到 length + 1，每次都要从头找尾，总复杂度会变成 O(n^2)。更好的做法是维护尾指针 r。

LinkList List_TailInsert(void) {
    int x;
    LinkList L = (LinkList)malloc(sizeof(LNode));
    if (L == NULL) {
        return NULL;
    }

    L->next = NULL;
    LNode *r = L;

    scanf("%d", &x);
    while (x != 999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return L;
        }

        s->data = x;
        s->next = NULL;

        r->next = s;
        r = s;

        scanf("%d", &x);
    }

    r->next = NULL;
    return L;
}
尾插核心：r->next = s; r = s;。必须保证 r 永远指向当前最后一个结点。

2. 头插法

头插法每次把新结点插到头结点后面，建立结果和输入顺序相反。

输入：10 16 27 结果：L -> 头结点 -> 27 -> 16 -> 10 -> NULL
LinkList List_HeadInsert(void) {
    int x;
    LinkList L = (LinkList)malloc(sizeof(LNode));
    if (L == NULL) {
        return NULL;
    }

    L->next = NULL;

    scanf("%d", &x);
    while (x != 999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return L;
        }

        s->data = x;
        s->next = L->next;
        L->next = s;

        scanf("%d", &x);
    }

    return L;
}
初始化不能忘：L->next = NULL 很重要。malloc 得到的空间可能有脏数据，如果不初始化，最后一个结点可能指向未知地址。

3. 头插法用于链表逆置

头插法天然得到逆序，所以常用来实现链表逆置。

void ReverseList(LinkList L) {
    LNode *p = L->next;
    LNode *r;

    L->next = NULL;

    while (p != NULL) {
        r = p->next;

        p->next = L->next;
        L->next = p;

        p = r;
    }
}
关键保护：先写 r = p->next 保存后继，因为马上要修改 p->next。不保存，后面的链就找不到了。

4. 不带头结点建表

不带头结点尾插

LinkList List_TailInsert_NoHead(void) {
    int x;
    LinkList L = NULL;
    LNode *r = NULL;

    scanf("%d", &x);
    while (x != 999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return L;
        }

        s->data = x;
        s->next = NULL;

        if (L == NULL) {
            L = s;
            r = s;
        } else {
            r->next = s;
            r = s;
        }

        scanf("%d", &x);
    }

    return L;
}

不带头结点头插

LinkList List_HeadInsert_NoHead(void) {
    int x;
    LinkList L = NULL;

    scanf("%d", &x);
    while (x != 999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        if (s == NULL) {
            return L;
        }

        s->data = x;
        s->next = L;
        L = s;

        scanf("%d", &x);
    }

    return L;
}
本部分重点：尾插法保持输入顺序，需要尾指针 r；头插法得到输入逆序，常用于链表逆置。建表整体复杂度都是 O(n)，但朴素尾插如果每次从头找尾会退化到 O(n^2)。

练习中心

练习 1：判断插入顺序

输入 10 16 27 999，分别用头插法和尾插法建表，结果是什么？

查看答案

尾插法：10 -> 16 -> 27，保持输入顺序。

头插法：27 -> 16 -> 10，得到输入逆序。

练习 2：指出代码错误

p->next = s;
s->next = p->next;

这两句用于插入新结点，有什么问题？

查看答案

顺序错了。执行第一句后 p->next 已经是 s，第二句会让 s->next 指向自己。正确顺序是：

s->next = p->next;
p->next = s;

练习 3：删除给定结点

只给一个结点指针 p，为什么不能用后继覆盖法删除尾结点？

查看答案

尾结点没有后继，p->next == NULL。后继覆盖法需要访问 q = p->next 的 q->data，此时 q 是空指针，会出错。只能从头找它的前驱。

练习 4：写出逆置核心

原地逆置单链表时，为什么要先写 r = p->next？

查看答案

因为后面要修改 p->next，如果不先保存原来的后继结点，后半段链表就找不到了。
