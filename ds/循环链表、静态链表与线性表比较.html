2.3.4 循环链表

一、循环链表的基本思想

循环链表是在普通链表的基础上做了一个小改进。

普通单链表中，最后一个结点的 next 指针指向 NULL。

C 语言
尾结点->next = NULL;

循环单链表中，最后一个结点的 next 指针不再指向 NULL，而是指回头结点。

C 语言
尾结点->next = 头结点;

也就是说，循环链表的首尾相接，形成一个环。

二、循环单链表

1. 循环单链表的定义

普通单链表：

循环单链表：

循环单链表与普通单链表的区别只有一个：

C 语言
普通单链表：尾结点->next == NULL
循环单链表：尾结点->next == head

2. 循环单链表的结点定义

C 语言
typedef struct LNode {
    int data;
    struct LNode *next;
} LNode;

其中：

C 语言
data：存放数据元素
next：指向下一个结点

3. 循环单链表的初始化

在初始化一个空的循环单链表时，需要让头结点的 next 指针指向头结点自己。

C 语言
head->next = head;

代码如下：

C 语言
#include <stdio.h>
#include <stdlib.h>

typedef struct LNode {
    int data;
    struct LNode *next;
} LNode;

LNode *initCircularList() {
    LNode *head = (LNode *)malloc(sizeof(LNode));

    if (head == NULL) {
        return NULL;
    }

    head->next = head;
    return head;
}

4. 循环单链表的判空

普通单链表为空时：

C 语言
head->next == NULL

循环单链表为空时：

C 语言
head->next == head

代码：

C 语言
int isEmpty(LNode *head) {
    return head->next == head;
}

5. 判断某个结点是否为表尾结点

普通单链表的表尾结点满足：

C 语言
p->next == NULL

循环单链表的表尾结点满足：

C 语言
p->next == head

代码：

C 语言
int isTail(LNode *head, LNode *p) {
    return p->next == head;
}

6. 循环单链表的遍历

普通单链表遍历时，可以用：

C 语言
while (p != NULL)

但是循环单链表不能这样写。

因为循环单链表中没有 NULL 作为结束标志，如果一直判断 p != NULL，循环就不会停止。

循环单链表的遍历应该判断是否重新回到头结点。

C 语言
void printList(LNode *head) {
    LNode *p = head->next;

    while (p != head) {
        printf("%d ", p->data);
        p = p->next;
    }

    printf("\n");
}

遍历停止条件是：

C 语言
p == head

7. 循环单链表的优点

普通单链表有一个限制：

如果只知道某个结点 p，只能访问它后面的结点，不能访问它前面的结点。

如果只知道 p，普通单链表无法找到 A。

但是循环单链表可以。

从 p 出发，沿着 next 一直走，最终可以绕回整个链表，所以可以访问链表中的任意结点。

8. 删除指定结点 p 的思路

如果要删除某个结点 p，需要先找到它的前驱结点。

删除的核心操作是：

C 语言
前驱结点->next = p->next;
free(p);

在循环单链表中，即使只知道 p，也可以沿着 next 一直找，直到找到某个结点 pre 满足：

C 语言
pre->next == p

这个 pre 就是 p 的前驱结点。

C 语言
void deleteNodeSafe(LNode *head, LNode *p) {
    if (head == NULL || p == NULL || p == head) {
        return;
    }

    LNode *pre = head;

    while (pre->next != head && pre->next != p) {
        pre = pre->next;
    }

    if (pre->next == p) {
        pre->next = p->next;
        free(p);
    }
}

三、循环单链表中尾指针的使用

在普通单链表中，如果只知道头结点，要找到尾结点，需要从头开始扫描。

时间复杂度是：

O(n)

但是对于循环单链表，可以让链表指针 L 不指向头结点，而是指向尾结点。

此时：

C 语言
L 是尾结点
L->next 是头结点
L->next->next 是第一个数据结点

这样可以在 O(1) 时间内找到：

C 语言
尾结点：L
头结点：L->next

如果经常在表头或表尾操作，可以用尾指针管理循环单链表。

注意：

如果在表尾插入或删除结点，需要及时修改 L 的指向。

四、循环双链表

1. 循环双链表的基本结构

普通双链表中：

C 语言
头结点->prior = NULL
尾结点->next = NULL

循环双链表中：

C 语言
尾结点->next = 头结点
头结点->prior = 尾结点

也就是说：

  • 所有 next 指针形成一个方向的环
  • 所有 prior 指针形成另一个方向的环

2. 循环双链表的结点定义

C 语言
typedef struct DNode {
    int data;
    struct DNode *prior;
    struct DNode *next;
} DNode;

其中：

C 语言
data：数据域
prior：指向前驱结点
next：指向后继结点

3. 循环双链表的初始化

空的循环双链表中，头结点的 prior 和 next 都指向自己。

C 语言
head->prior = head;
head->next = head;

代码：

C 语言
DNode *initCircularDList() {
    DNode *head = (DNode *)malloc(sizeof(DNode));

    if (head == NULL) {
        return NULL;
    }

    head->prior = head;
    head->next = head;

    return head;
}

4. 循环双链表的判空

循环双链表为空时：

C 语言
head->next == head

代码：

C 语言
int isDListEmpty(DNode *head) {
    return head->next == head;
}

也可以通过 head->prior == head 判断，但通常判断 head->next == head 即可。

5. 判断某个结点是否为表尾结点

循环双链表的表尾结点满足：

C 语言
p->next == head

代码：

C 语言
int isDListTail(DNode *head, DNode *p) {
    return p->next == head;
}

注意：

当循环双链表为空时，头结点的 next 也指向自己，所以从逻辑上看，头结点既像第一个结点，也像最后一个结点。

五、循环双链表的插入和删除

1. 在结点 p 后插入结点 s

普通双链表中，如果 p 是尾结点，那么：

C 语言
p->next == NULL

如果执行：

C 语言
p->next->prior = s;

就会访问空指针，程序出错。

但是循环双链表中，即使 p 是尾结点：

C 语言
p->next == head

所以 p->next->prior = s 修改的是头结点的 prior，这是正确的。

插入前：

插入后：

代码：

C 语言
void insertAfter(DNode *p, DNode *s) {
    s->next = p->next;
    p->next->prior = s;
    s->prior = p;
    p->next = s;
}

四句代码含义：

C 语言
s->next = p->next;

让 s 指向 p 原来的后继。

C 语言
p->next->prior = s;

让原后继的前驱改成 s。

C 语言
s->prior = p;

让 s 的前驱指向 p。

C 语言
p->next = s;

最后让 p 的后继改成 s。

注意：

p->next = s 要放在最后，否则会丢失 p 原来的后继结点。

2. 删除结点 q

假设 p 是 q 的前驱：

C 语言
p->next == q

删除前：

删除后：

代码：

C 语言
void deleteAfter(DNode *p) {
    DNode *q = p->next;

    p->next = q->next;
    q->next->prior = p;

    free(q);
}

如果 q 是普通双链表的尾结点，则 q->next == NULL，执行：

C 语言
q->next->prior = p;

会出错。

但循环双链表中，如果 q 是尾结点：

C 语言
q->next == head

所以：

C 语言
q->next->prior = p;

其实是在修改头结点的 prior，逻辑正确。

3. 更常用的删除指定结点 q

如果已经知道要删除的结点 q，循环双链表删除很方便：

C 语言
void deleteDNode(DNode *head, DNode *q) {
    if (head == NULL || q == NULL || q == head) {
        return;
    }

    q->prior->next = q->next;
    q->next->prior = q->prior;

    free(q);
}

因为循环双链表没有 NULL 边界，所以不需要额外判断 q 是不是尾结点。

六、循环双链表完整代码

C 语言
#include <stdio.h>
#include <stdlib.h>

typedef struct DNode {
    int data;
    struct DNode *prior;
    struct DNode *next;
} DNode;

DNode *initList() {
    DNode *head = (DNode *)malloc(sizeof(DNode));

    if (head == NULL) {
        return NULL;
    }

    head->prior = head;
    head->next = head;

    return head;
}

int isEmpty(DNode *head) {
    return head->next == head;
}

DNode *createNode(int data) {
    DNode *node = (DNode *)malloc(sizeof(DNode));

    if (node == NULL) {
        return NULL;
    }

    node->data = data;
    node->prior = NULL;
    node->next = NULL;

    return node;
}

void insertAfter(DNode *p, DNode *s) {
    s->next = p->next;
    p->next->prior = s;
    s->prior = p;
    p->next = s;
}

void deleteNode(DNode *head, DNode *q) {
    if (head == NULL || q == NULL || q == head) {
        return;
    }

    q->prior->next = q->next;
    q->next->prior = q->prior;

    free(q);
}

void printList(DNode *head) {
    DNode *p = head->next;

    while (p != head) {
        printf("%d ", p->data);
        p = p->next;
    }

    printf("\n");
}

void printReverse(DNode *head) {
    DNode *p = head->prior;

    while (p != head) {
        printf("%d ", p->data);
        p = p->prior;
    }

    printf("\n");
}

int main() {
    DNode *head = initList();

    DNode *a = createNode(10);
    DNode *b = createNode(20);
    DNode *c = createNode(30);

    insertAfter(head, a);
    insertAfter(a, b);
    insertAfter(b, c);

    printf("正向遍历：");
    printList(head);

    printf("反向遍历：");
    printReverse(head);

    deleteNode(head, b);

    printf("删除 20 后：");
    printList(head);

    return 0;
}

七、所有链表代码的通用检查点

写链表代码时，不管是单链表、双链表，还是循环链表，都要关注以下问题：

1. 空表是什么状态

普通单链表：

C 语言
head->next == NULL

循环单链表：

C 语言
head->next == head

普通双链表：

C 语言
head->next == NULL
head->prior == NULL

循环双链表：

C 语言
head->next == head
head->prior == head

知道空表状态，就知道怎么初始化。

2. 怎么判断表尾

普通链表：

C 语言
p->next == NULL

循环链表：

C 语言
p->next == head

3. 怎么遍历

普通链表：

C 语言
while (p != NULL)

循环链表：

C 语言
while (p != head)

4. 插入删除要考虑边界

写链表插入、删除时，要考虑：

C 语言
表头
表中
表尾
空表

普通链表在表尾处经常需要特殊处理，因为尾结点后面是 NULL。

循环链表的优势是：

C 语言
尾结点后面不是 NULL，而是头结点

所以很多操作可以统一处理。

循环链表重点总结

循环单链表：

C 语言
尾结点->next = head;
head->next = head;  // 空表

循环双链表：

C 语言
尾结点->next = head;
head->prior = 尾结点;

head->next = head;
head->prior = head; // 空表

循环链表遍历停止条件不是 NULL，而是回到头结点：

C 语言
p != head

判断尾结点：

C 语言
p->next == head

循环双链表插入核心代码：

C 语言
s->next = p->next;
p->next->prior = s;
s->prior = p;
p->next = s;

循环双链表删除核心代码：

C 语言
q->prior->next = q->next;
q->next->prior = q->prior;
free(q);

考点速记

循环单链表空表

head->next == head，尾结点条件也是 p->next == head。

循环双链表空表

head->next == head 且 head->prior == head。

遍历停止

循环链表不能等 NULL，要判断是否回到头结点。

常见陷阱

  • 把循环链表遍历写成 while (p != NULL)，会导致无法正常停止。
  • 循环双链表后插时过早执行 p->next = s，会丢失原后继。
  • 删除结点后忘记 free，会造成堆空间泄漏。

答题模板

说明循环链表时，先写普通链表尾指针指向 NULL，再对比循环链表尾指针指回头结点，最后补充空表、判尾和遍历条件。

代码检查清单

初始化是否让头结点指向自己遍历条件是否是 p != head插入指针顺序是否保留原后继删除前是否排除头结点

2.3.5 静态链表

一、什么是静态链表

静态链表本质上是：

C 语言
用数组模拟链表

普通单链表中的各个结点可以离散地分布在内存中，每个结点存放：

C 语言
数据元素 data
指向下一个结点的指针 next

静态链表则是先分配一整片连续的内存空间，也就是一个数组。

数组中的每一个元素就是一个结点。

静态链表中的每个结点存放：

C 语言
数据元素 data
下一个结点的数组下标 next

这里的 next 不是真正的地址，而是数组下标，也叫游标。

二、静态链表和单链表的区别

单链表：

C 语言
next 存放的是下一个结点的内存地址

静态链表：

C 语言
next 存放的是下一个结点的数组下标

单链表示意：

静态链表示意：

数组下标datanext
0不存实际数据2
1B3
2A1
3C-1
4空闲-2
5空闲-2

它表示的逻辑链表是：

注意：

数组的物理顺序是：

C 语言
a[0], a[1], a[2], a[3], a[4], ...

链表的逻辑顺序是：

C 语言
a[0] -> a[2] -> a[1] -> a[3]

所以静态链表虽然底层是数组，但逻辑上仍然是链表。

三、静态链表中的特殊值

文稿中涉及两个重要特殊值：

C 语言
-1：表示链表结束，相当于单链表中的 NULL
-2：表示该数组位置空闲，可以用来存放新结点

例如：

C 语言
a[0].next == -1

表示静态链表为空。

C 语言
a[i].next == -2

表示 a[i] 这个位置是空闲结点。

四、静态链表的地址计算

静态链表的底层是数组，数组元素在内存中连续存放。

假设：

C 语言
每个结点大小 = 8 字节
数组起始地址 = base

那么数组下标为 2 的结点地址是：

C 语言
base + 2 * 8

一般公式：

C 语言
某结点地址 = 起始地址 + 数组下标 * 每个结点大小

这说明数组下标可以映射成实际内存地址。

但是注意：

静态链表的“按位序查找”和数组的“按下标访问”不是一回事。

位序是逻辑顺序，数组下标是物理位置。

第 3 个数据结点：a[3]

        cur = a[cur].next;
        count++;

    if (cur != END && count == i) {

十一、静态链表的特点

静态链表的本质：

C 语言
数组 + 游标

它和顺序表一样，底层使用连续空间。

但它和链表一样，逻辑相邻的元素不要求物理相邻。

1. 静态链表的优点

插入和删除时，不需要像顺序表那样大量移动元素。

只需要修改相关结点的 next 游标。

C 语言
a[freeIndex].next = a[pre].next;
a[pre].next = freeIndex;

删除时：

C 语言
a[pre].next = a[del].next;
a[del].next = FREE;

2. 静态链表的缺点

静态链表不支持随机存取。

虽然底层是数组，但要找第 i 个逻辑元素，仍然要从头结点开始沿着 next 查找。

时间复杂度：

O(n)

静态链表容量固定。

一旦声明：

C 语言
Node a[MAX_SIZE];

最多能存多少结点就已经确定了，不能自动扩容。

3. 静态链表的适用场景

静态链表现在使用相对较少。

它主要适用于：

C 语言
早期不支持指针的低级语言
数据元素数量基本固定的场景

文稿中提到：

操作系统文件管理中的 FAT，也就是文件分配表，本质上就有静态链表的思想。

可以理解为：

C 语言
用表项中的下标关系表示文件块之间的连接关系

静态链表重点总结

静态链表就是：

C 语言
用数组实现链表

每个结点：

C 语言
typedef struct Node {
    int data;
    int next;
} Node;

其中：

C 语言
data：存放数据
next：存放下一个结点的数组下标

特殊规则：

C 语言
a[0] 是头结点，不存实际数据
next == -1 表示链表结束
next == -2 表示该位置空闲

初始化：

C 语言
a[0].next = -1;

for (int i = 1; i < MAX_SIZE; i++) {
    a[i].next = -2;
}

插入核心：

C 语言
a[freeIndex].next = a[pre].next;
a[pre].next = freeIndex;

删除核心：

C 语言
a[pre].next = a[del].next;
a[del].next = -2;

重点理解：

C 语言
静态链表底层是数组，但逻辑上是链表。
它用数组下标模拟指针，用游标维护元素之间的逻辑关系。

考点速记

本质

静态链表是“数组 + 游标”，用数组下标模拟指针。

特殊值

-1 表示结束，-2 表示空闲位置。

位序区别

逻辑位序要沿 next 查，不能直接等同数组下标。

常见陷阱

  • 误以为第 2 个元素就是 a[2]，忽略逻辑链顺序。
  • 把游标当真实指针理解；游标只是数组下标。
  • 插入后忘记把新结点从空闲状态改成链中状态。

答题模板

解释静态链表时，按“数组连续存储、结点含 data 和 next、next 存下标、-1/-2 标记状态、逻辑顺序由游标决定”展开。

代码检查清单

头结点是否用 a[0]空表是否 a[0].next = -1空闲结点是否标成 -2删除后是否回收为空闲

2.3.6 顺序表和链表的比较

一、用数据结构三要素比较

学习一个数据结构，要关注三个方面：

C 语言
逻辑结构
物理结构
数据运算

也就是：

C 语言
它在逻辑上是什么关系？
它在内存中怎么存？
它支持哪些操作？效率如何？

顺序表和链表都属于线性表。

它们的逻辑结构相同，都是线性结构。

线性结构的特点：

C 语言
除第一个元素外，每个元素有唯一前驱
除最后一个元素外，每个元素有唯一后继

示意图：

顺序表和链表真正的区别在于：

C 语言
物理存储方式不同
基本操作效率不同

二、存储结构比较

1. 顺序表的存储结构

顺序表采用顺序存储。

系统给它分配一整片连续的内存空间。

因为每个数据元素大小相等，所以只要知道起始地址，就可以快速算出任意元素的位置。

地址计算公式：

C 语言
第 i 个元素地址 = 起始地址 + i * 每个元素大小

因此顺序表支持随机存取。

按位查找时间复杂度：

O(1)

顺序表的优点

顺序表中的每个位置只存数据本身，不需要额外存指针。

所以存储密度高。

C 语言
int a[5] = {10, 20, 30, 40, 50};

每个位置只存一个整数，没有额外指针域。

顺序表的缺点

顺序表要求连续存储空间。

如果一开始分配太小，后面扩容麻烦。

如果一开始分配太大，长期空闲空间多，会浪费内存。

所以顺序表容量调整不方便。

2. 链表的存储结构

链表采用链式存储。

每个结点可以离散地分布在内存中的不同位置。

单链表结点：

C 语言
typedef struct LNode {
    int data;
    struct LNode *next;
} LNode;

示意图：

链表不要求连续空间。

每次要添加新结点时，可以用 malloc 动态申请一小片空间。

C 语言
LNode *p = (LNode *)malloc(sizeof(LNode));

链表的优点

链表扩容方便。

因为它不要求一整片连续空间，只需要每次申请一个新结点，然后用指针连入链表。

C 语言
新结点->next = 原来的后继;
前驱结点->next = 新结点;

所以链表存储容量更灵活。

链表的缺点

链表不能随机存取。

如果要找第 i 个元素，只能从第一个结点开始依次往后找。

C 语言
LNode *p = head->next;

while (...) {
    p = p->next;
}

按位查找时间复杂度：

O(n)

另外，每个链表结点除了存数据，还要存指针。

所以链表存储密度较低。

单链表结点：

C 语言
data + next

双链表结点：

C 语言
prior + data + next

都比顺序表多了指针空间。

三、存储结构总体对比

对比点顺序表链表
存储方式连续存储链式离散存储
随机存取支持，O(1)不支持，O(n)
存储密度高低
容量调整不方便比较方便
额外空间基本没有需要指针域

核心记忆：

C 语言
顺序表查找强，链表增删灵活。

C 语言
创、销、增、删、改、查

即：

C 语言
创建

增加

五、创建操作比较

1. 顺序表的创建

顺序表初始化时，需要提前分配一整片连续空间。

静态分配：

C 语言
#define MAX_SIZE 100

typedef struct {
    int data[MAX_SIZE];
    int length;
} SqList;

void initList(SqList *L) {
    L->length = 0;
}

这种方式容量固定。

C 语言
MAX_SIZE = 100

表示最多存 100 个元素。

如果空间太小，后面不够用。

如果空间太大，可能浪费内存。

2. 动态分配顺序表

C 语言
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    int *data;
    int length;
    int capacity;
} SeqList;

void initSeqList(SeqList *L, int capacity) {
    L->data = (int *)malloc(sizeof(int) * capacity);

    if (L->data == NULL) {
        printf("内存分配失败\n");
        exit(1);
    }

    L->length = 0;
    L->capacity = capacity;
}

动态分配的顺序表容量可以改变。

但是扩容仍然麻烦，通常需要：

C 语言
申请更大的新空间
复制旧数据
释放旧空间

所以动态顺序表虽然比静态顺序表灵活，但仍不如链表灵活。

3. 链表的创建

链表初始化通常只需要一个头指针，或者申请一个头结点。

带头结点的单链表：

C 语言
typedef struct LNode {
    int data;
    struct LNode *next;
} LNode;

LNode *initList() {
    LNode *head = (LNode *)malloc(sizeof(LNode));

    if (head == NULL) {
        return NULL;
    }

    head->next = NULL;
    return head;
}

链表之后需要增加元素时，再用 malloc 一个一个申请新结点。

创建操作总结

顺序表创建时：

C 语言
需要提前考虑容量
容量太小，扩容麻烦
容量太大，浪费空间

链表创建时：

C 语言
只需要头指针或头结点
需要新元素时再 malloc
容量更灵活

因此，从存储容量弹性来看：

C 语言
链表更胜一筹

六、销毁操作比较

销毁操作在考试中不一定经常考，但实际写代码非常重要。

1. 顺序表的销毁

如果顺序表是静态数组实现：

C 语言
typedef struct {
    int data[100];
    int length;
} SqList;

销毁时通常只需要：

C 语言
L.length = 0;

这里的 length = 0 只是逻辑上把顺序表标记为空表。

静态数组的空间会在生命周期结束后由系统自动回收。

如果顺序表是动态数组实现：

C 语言
typedef struct {
    int *data;
    int length;
    int capacity;
} SeqList;

销毁时必须手动释放：

C 语言
void destroySeqList(SeqList *L) {
    free(L->data);
    L->data = NULL;
    L->length = 0;
    L->capacity = 0;
}

因为 malloc 申请的是堆区空间，不会自动释放。

原则：

C 语言
malloc 和 free 要成对出现

2. 链表的销毁

链表中的每个结点通常都是用 malloc 申请的。

所以销毁链表时，要依次释放每个结点。

C 语言
void destroyList(LNode *head) {
    LNode *p = head;
    LNode *nextNode;

    while (p != NULL) {
        nextNode = p->next;
        free(p);
        p = nextNode;
    }
}

注意不能这样写：

C 语言
free(p);
p = p->next;

因为 p 被释放后，再访问 p->next 是不安全的。

正确做法是：

C 语言
先保存下一个结点
再释放当前结点

3. 关于 malloc 和 free

普通变量或数组：

C 语言
int x;
int a[100];

它们的空间通常会由系统自动管理，生命周期结束后自动回收。

malloc 申请的空间：

C 语言
int *p = malloc(sizeof(int) * 100);

需要手动释放：

C 语言
free(p);

链表：

C 语言

如果一个数据元素非常大，比如一个元素占 1MB，那么顺序表移动大量元素的代价会很高。

八、修改和查找操作比较

链表：

所以修改操作的关键不是“改”，而是“查”。

查找快，修改就快。

按值查找就是：

C 语言
查找值等于 x 的元素

顺序表无序时

如果顺序表中的数据无序，只能从头到尾依次比较。

O(n)

如果顺序表中的元素有序，可以使用折半查找，也叫二分查找。

时间复杂度：

O(log n)

顺序表能用折半查找，是因为它可以快速访问中间元素。

因为链表不能直接访问中间结点。

要找中间结点，仍然需要从头走过去。

所以链表按值查找，无论有序还是无序，通常都是：

O(n)

十、什么时候用顺序表，什么时候用链表

1. 适合使用链表的场景

如果线性表的长度难以估计，并且经常插入、删除，适合使用链表。

文稿例子：

C 语言
奶茶店排队取号或叫号系统

原因：

C 语言
不知道今天会有多少顾客
新顾客取号时需要插入元素
顾客取餐离开时需要删除元素

这种场景：

C 语言
表长不稳定
增删频繁

所以链表更合适。

2. 适合使用顺序表的场景

如果线性表长度可预估、比较稳定，并且查询操作多，适合使用顺序表。

文稿例子：

C 语言
课堂点名系统

原因：

C 语言
一个班级学生人数基本固定
一个学期内变化不大
点名时主要是查询学生信息

这种场景：

C 语言
表长稳定
查询较多

所以顺序表更合适。

十一、关键代码整理

1. 动态顺序表创建和销毁

C 语言
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    int *data;
    int length;
    int capacity;
} SeqList;

void initSeqList(SeqList *L, int capacity) {
    L->data = (int *)malloc(sizeof(int) * capacity);

    if (L->data == NULL) {
        printf("内存分配失败\n");
        exit(1);
    }

    L->length = 0;
    L->capacity = capacity;
}

void destroySeqList(SeqList *L) {
    free(L->data);
    L->data = NULL;
    L->length = 0;
    L->capacity = 0;
}

2. 顺序表插入和删除

C 语言
int insertSeqList(SeqList *L, int pos, int value) {
    if (pos < 1 || pos > L->length + 1) {
        return 0;
    }

    if (L->length >= L->capacity) {
        return 0;
    }

    for (int i = L->length; i >= pos; i--) {
        L->data[i] = L->data[i - 1];
    }

    L->data[pos - 1] = value;
    L->length++;

    return 1;
}

int deleteSeqList(SeqList *L, int pos) {
    if (pos < 1 || pos > L->length) {
        return 0;
    }

    for (int i = pos; i < L->length; i++) {
        L->data[i - 1] = L->data[i];
    }

    L->length--;

    return 1;
}

3. 链表创建、插入、删除、销毁

C 语言
#include <stdio.h>
#include <stdlib.h>

typedef struct LNode {
    int data;
    struct LNode *next;
} LNode;

LNode *initList() {
    LNode *head = (LNode *)malloc(sizeof(LNode));

    if (head == NULL) {
        printf("内存分配失败\n");
        exit(1);
    }

    head->next = NULL;
    return head;
}

void insertAfter(LNode *p, int value) {
    LNode *s = (LNode *)malloc(sizeof(LNode));

    if (s == NULL) {
        printf("内存分配失败\n");
        exit(1);
    }

    s->data = value;
    s->next = p->next;
    p->next = s;
}

void deleteAfter(LNode *p) {
    if (p == NULL || p->next == NULL) {
        return;
    }

    LNode *q = p->next;
    p->next = q->next;
    free(q);
}

void destroyList(LNode *head) {
    LNode *p = head;
    LNode *nextNode;

    while (p != NULL) {
        nextNode = p->next;
        free(p);
        p = nextNode;
    }
}

十二、考试答题框架

遇到“比较顺序表和链表”这种开放题，可以按照以下框架答：

C 语言
逻辑结构
存储结构

示例答法：

第一，逻辑结构上，顺序表和链表都是线性表，数据元素之间都是一对一关系。

第二，存储结构上，顺序表采用连续存储，支持随机存取，存储密度高，但容量调整不方便；链表采用链式存储，结点可以离散分布，扩容灵活，但不能随机存取，并且需要额外指针空间。

第三，基本操作上，顺序表按位查找效率高，为 O(1)；链表按位查找需要从头遍历，为 O(n)。顺序表插入删除需要移动大量元素；链表插入删除主要修改指针，但通常也需要先查找位置。

第四，应用场景上，如果表长稳定且查询操作多，适合顺序表；如果表长难以估计且插入删除频繁，适合链表。

顺序表和链表比较重点总结

顺序表：

C 语言
连续存储
支持随机存取
按位查找 O(1)
插入删除需要移动元素
存储密度高
容量调整不方便

链表：

C 语言
链式存储
不支持随机存取
查找 O(n)
插入删除主要修改指针
扩容灵活
需要额外指针空间

使用选择：

C 语言
查询多、表长稳定：用顺序表
增删多、表长不确定：用链表

核心理解：

C 语言
顺序表牺牲扩容灵活性，换来了快速随机访问。
链表牺牲随机访问能力，换来了更灵活的插入、删除和空间分配。

考点速记

场景优先选择原因
表长稳定、查询频繁顺序表支持随机存取，按位查找为 O(1)
表长难估、增删频繁链表无需连续空间，已知前驱时插删本身为 O(1)
需要折半查找有序顺序表可以快速访问中间元素，时间复杂度 O(log n)

常见陷阱

  • 只看大 O，忽略顺序表是“移动元素”，链表是“查找位置”。
  • 认为链表有序就能高效二分，实际上链表不能随机访问中间结点。
  • 动态顺序表虽然能扩容，但扩容通常仍要申请新空间、复制、释放旧空间。
