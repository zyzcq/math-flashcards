双链表：定义、插删、销毁、遍历

这份笔记围绕文稿完整整理：双链表为什么出现、结点如何定义、带头结点如何初始化、插入删除怎样维护两条链、如何销毁、如何前后遍历，以及空指针和复杂度问题。

学习地图 概念与定义 初始化 插入 删除与销毁 遍历查找 完整代码 练习 复习清单

学习地图

总主线：双链表是在单链表的基础上多加 prior 指针，让每个结点既能找到后继，也能找到前驱。

先看结构

单链表只有 next，双链表有 prior 和 next 两个指针域。

再看改链

插入和删除时不能只改后向链，还要同步维护前向链。

最后看边界

插入表尾、删除尾结点时，必须避免访问空指针。

复习口诀：双链表多了前驱，找前驱变容易；插删要改两边，尾部别碰空指针。
一、为什么需要双链表

1. 单链表的问题

单链表每个结点只有指向后继结点的指针。如果给定一个结点 p，想找它的后继很容易，写 p->next 就行；但想找它的前驱很麻烦，因为结点里没有保存“前一个结点是谁”。

单链表： [10|next] -> [20|next] -> [30|NULL] 给定 p 指向 20： p->next 可以找到 30 但 p 的前驱 10 不能直接找到

2. 双链表的解决办法

双链表在单链表基础上增加一个指针域 prior，让它指向当前结点的前驱结点。

双链表： NULL <- [头结点] <-> [10] <-> [20] <-> [30] -> NULL 每个数据结点： prior 指向前驱 next 指向后继

3. 结点定义

文稿中把双链表结点命名为 DNode，其中 D 来自 double，表示双向。

#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

typedef int ElemType;

typedef struct DNode {
    ElemType data;
    struct DNode *prior;
    struct DNode *next;
} DNode, *DLinkList;
成员 作用
data 存放数据元素
prior 指向前驱结点
next 指向后继结点

4. DNode 和 DLinkList

和单链表类似，DNode * 和 DLinkList 本质都是 struct DNode *，只是语义不同。

写法 本质 强调
DNode *p 双链表结点指针 p 是某个结点
DLinkList L 双链表结点指针 L 是一条链表的入口
本部分重点：双链表的核心变化是增加 prior。它解决单链表找前驱麻烦的问题，但后面插入删除时也要多维护一条前向链。
二、带头结点双链表的初始化与判空

1. 初始化

文稿讨论的是带头结点的双链表。初始化时，先申请头结点空间，再让头结点的 prior 和 next 都指向 NULL。

初始化后： L ↓ [prior=NULL | 头结点 | next=NULL]
bool InitDLinkList(DLinkList *L) {
    *L = (DNode *)malloc(sizeof(DNode));
    if (*L == NULL) {
        return false;
    }

    (*L)->prior = NULL;
    (*L)->next = NULL;
    return true;
}
为什么头结点的 prior 永远是 NULL：头结点前面不会再有其他结点，所以它没有前驱。

2. 判空

带头结点的双链表即使为空，也有一个头结点。判断是否为空，要看头结点之后有没有数据结点。

bool Empty(DLinkList L) {
    return L->next == NULL;
}
空表： L -> [头结点] -> NULL 判空： L->next == NULL
本部分重点：初始化要设置 L->prior = NULL 和 L->next = NULL。带头结点双链表判空看 L->next，不是看 L。
三、双链表插入：在 p 后插入 s

1. 插入目标

课本中的插入是后插操作：给定结点 p 和新结点 s，把 s 插到 p 后面。

插入前： [P] <----> [B] 插入后： [P] <----> [S] <----> [B]

2. 课本基本思路

普通情况下要改四条指针：

  1. s->next = p->next：让 s 指向原来 p 的后继。
  2. p->next->prior = s：让原后继的前驱改成 s。
  3. s->prior = p：让 s 的前驱指向 p。
  4. p->next = s：让 p 的后继改成 s。

3. 更严谨的代码

如果 p 刚好是最后一个结点，那么 p->next == NULL。这时不能执行 p->next->prior = s，否则会空指针错误。

bool InsertNextDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL) {
        return false;
    }

    s->next = p->next;

    if (p->next != NULL) {
        p->next->prior = s;
    }

    s->prior = p;
    p->next = s;

    return true;
}
尾部插入的边界：当 p 是尾结点时，p->next == NULL，只需要让 s->next = NULL、s->prior = p、p->next = s。

4. 指针修改顺序不能乱

文稿中特别提醒：如果先执行 p->next = s，再执行 s->next = p->next，就会让 s->next 指向 s 自己。

错误顺序： p->next = s; s->next = p->next; 结果： [P] -> [S] ↑ | └──┘ s->next 指向自己，后面的链断掉

5. 后插可以推广到其他插入

按位插入

先从头结点开始找到第 i - 1 个结点，再对它执行后插。

前插操作

双链表能直接通过 p->prior 找到前驱，所以在 p 前插入，可以转化为对 p->prior 后插。

bool ListInsert(DLinkList L, int i, ElemType e) {
    if (i < 1) {
        return false;
    }

    DNode *p = L;
    int j = 0;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL) {
        return false;
    }

    DNode *s = (DNode *)malloc(sizeof(DNode));
    if (s == NULL) {
        return false;
    }

    s->data = e;
    return InsertNextDNode(p, s);
}
本部分重点：双链表后插要维护四个方向关系。最容易错的是尾部空指针和指针修改顺序。先保存旧后继，再建立新连接。
四、双链表删除与销毁

1. 删除 p 的后继结点

文稿讨论的是删除指定结点 p 的后继结点。令 q = p->next，那么真正要删除的是 q。

删除前： [P] <----> [Q] <----> [B] 删除后： [P] <---------------> [B] 最后 free(Q)

2. 严谨代码

如果 p 没有后继，删除失败；如果 q 是尾结点，不能访问 q->next->prior。

bool DeleteNextDNode(DNode *p) {
    if (p == NULL) {
        return false;
    }

    DNode *q = p->next;
    if (q == NULL) {
        return false;
    }

    p->next = q->next;

    if (q->next != NULL) {
        q->next->prior = p;
    }

    free(q);
    return true;
}
删除尾结点的边界：如果 q 是最后一个结点，q->next == NULL，这时只需要 p->next = NULL，不能再写 q->next->prior = p。

3. 按位删除

按位删除可以先找第 i - 1 个结点，再删除它的后继。

bool ListDelete(DLinkList L, int i, ElemType *e) {
    if (i < 1) {
        return false;
    }

    DNode *p = L;
    int j = 0;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL || p->next == NULL) {
        return false;
    }

    *e = p->next->data;
    return DeleteNextDNode(p);
}

4. 销毁双链表

销毁双链表时，可以用循环不断删除头结点的后继结点，直到只剩头结点。最后释放头结点，并让头指针指向 NULL。

void DestroyDLinkList(DLinkList *L) {
    if (L == NULL || *L == NULL) {
        return;
    }

    while ((*L)->next != NULL) {
        DeleteNextDNode(*L);
    }

    free(*L);
    *L = NULL;
}
销毁过程： L -> 头结点 <-> 10 <-> 20 <-> 30 -> NULL L -> 头结点 <-> 20 <-> 30 -> NULL L -> 头结点 <-> 30 -> NULL L -> 头结点 -> NULL free(头结点) L = NULL
本部分重点：删除时先保存目标 q，再让 p 跳过 q，如果 q 有后继，再修正后继的 prior，最后 free(q)。销毁要释放所有数据结点和头结点。
五、遍历与查找

1. 后向遍历

后向遍历就是沿着 next 从前往后走。循环内部可以打印数据，也可以做其他处理。

void PrintForward(DLinkList L) {
    DNode *p = L->next;

    while (p != NULL) {
        printf("%d ", p->data);
        p = p->next;
    }

    printf("\n");
}

2. 前向遍历

前向遍历就是沿着 prior 从后往前走。如果不想处理头结点，可以让循环在 p->prior == NULL 时停止。

void PrintBackward(DNode *p) {
    while (p != NULL && p->prior != NULL) {
        printf("%d ", p->data);
        p = p->prior;
    }

    printf("\n");
}
为什么用 p->prior != NULL：头结点的 prior 是 NULL，这个条件可以避免把头结点当作数据结点处理。

3. 按位查找

双链表虽然能双向遍历，但依旧不支持随机存取。按位查找仍要一个一个走。

DNode *GetElem(DLinkList L, int i) {
    if (i < 0) {
        return NULL;
    }

    DNode *p = L;
    int j = 0;

    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }

    return p;
}

4. 按值查找

DNode *LocateElem(DLinkList L, ElemType e) {
    DNode *p = L->next;

    while (p != NULL && p->data != e) {
        p = p->next;
    }

    return p;
}

5. 时间复杂度

操作 复杂度 原因
已知 p 找前驱 O(1) 直接访问 p->prior
已知 p 找后继 O(1) 直接访问 p->next
在 p 后插入 O(1) 只改有限条指针
删除 p 后继 O(1) 只改有限条指针
按位查找 O(n) 不支持随机存取，只能遍历
按值查找 O(n) 需要逐个结点比较
销毁整表 O(n) 每个结点都要释放一次
本部分重点：双链表能向前、向后遍历，但仍不能像数组那样随机访问。因此按位查找和按值查找仍然是 O(n)。

完整代码整理

下面是一份带头结点双链表的完整 C 代码，包含文稿里的初始化、判空、插入、删除、销毁、遍历和查找。

#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

typedef int ElemType;

typedef struct DNode {
    ElemType data;
    struct DNode *prior;
    struct DNode *next;
} DNode, *DLinkList;

bool InitDLinkList(DLinkList *L) {
    *L = (DNode *)malloc(sizeof(DNode));
    if (*L == NULL) {
        return false;
    }

    (*L)->prior = NULL;
    (*L)->next = NULL;
    return true;
}

bool Empty(DLinkList L) {
    return L->next == NULL;
}

bool InsertNextDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL) {
        return false;
    }

    s->next = p->next;

    if (p->next != NULL) {
        p->next->prior = s;
    }

    s->prior = p;
    p->next = s;
    return true;
}

bool ListInsert(DLinkList L, int i, ElemType e) {
    if (i < 1) {
        return false;
    }

    DNode *p = L;
    int j = 0;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL) {
        return false;
    }

    DNode *s = (DNode *)malloc(sizeof(DNode));
    if (s == NULL) {
        return false;
    }

    s->data = e;
    return InsertNextDNode(p, s);
}

bool DeleteNextDNode(DNode *p) {
    if (p == NULL) {
        return false;
    }

    DNode *q = p->next;
    if (q == NULL) {
        return false;
    }

    p->next = q->next;

    if (q->next != NULL) {
        q->next->prior = p;
    }

    free(q);
    return true;
}

bool ListDelete(DLinkList L, int i, ElemType *e) {
    if (i < 1) {
        return false;
    }

    DNode *p = L;
    int j = 0;

    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }

    if (p == NULL || p->next == NULL) {
        return false;
    }

    *e = p->next->data;
    return DeleteNextDNode(p);
}

DNode *GetElem(DLinkList L, int i) {
    if (i < 0) {
        return NULL;
    }

    DNode *p = L;
    int j = 0;

    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }

    return p;
}

DNode *LocateElem(DLinkList L, ElemType e) {
    DNode *p = L->next;

    while (p != NULL && p->data != e) {
        p = p->next;
    }

    return p;
}

void PrintForward(DLinkList L) {
    DNode *p = L->next;

    while (p != NULL) {
        printf("%d ", p->data);
        p = p->next;
    }

    printf("\n");
}

void PrintBackward(DNode *p) {
    while (p != NULL && p->prior != NULL) {
        printf("%d ", p->data);
        p = p->prior;
    }

    printf("\n");
}

void DestroyDLinkList(DLinkList *L) {
    if (L == NULL || *L == NULL) {
        return;
    }

    while ((*L)->next != NULL) {
        DeleteNextDNode(*L);
    }

    free(*L);
    *L = NULL;
}

int main(void) {
    DLinkList L;

    if (!InitDLinkList(&L)) {
        printf("初始化失败\n");
        return 1;
    }

    ListInsert(L, 1, 10);
    ListInsert(L, 2, 20);
    ListInsert(L, 3, 30);

    printf("后向遍历：");
    PrintForward(L);

    DNode *p = LocateElem(L, 30);
    if (p != NULL) {
        printf("从 30 开始前向遍历：");
        PrintBackward(p);
    }

    ElemType deleted;
    if (ListDelete(L, 2, &deleted)) {
        printf("删除第 2 个结点：%d\n", deleted);
    }

    printf("删除后：");
    PrintForward(L);

    DestroyDLinkList(&L);
    return 0;
}

练习中心

练习 1：为什么双链表找前驱方便

给定结点 p，单链表和双链表分别怎么找它的前驱？

查看答案

单链表不能从 p 直接找前驱，通常要从头结点开始遍历。双链表可以直接写 p->prior，时间复杂度是 O(1)。

练习 2：指出插入代码问题

p->next = s;
s->next = p->next;

这两句如果用于后插，有什么问题？

查看答案

顺序错了。第一句执行后，p->next 已经变成 s，第二句会让 s->next 指向 s 自己。正确做法是先保存原后继：s->next = p->next。

练习 3：尾结点边界

为什么插入时要判断 p->next != NULL？删除时要判断 q->next != NULL？

查看答案

如果 p 是尾结点，p->next 是空指针，不能访问 p->next->prior。如果 q 是尾结点，q->next 是空指针，不能访问 q->next->prior。

练习 4：遍历条件

前向遍历时，如果只想处理数据结点，为什么可以用 p->prior != NULL 控制循环？

查看答案

因为带头结点双链表中，头结点的 prior 是 NULL。当 p->prior == NULL 时，说明 p 已经是头结点，不应该再处理。
