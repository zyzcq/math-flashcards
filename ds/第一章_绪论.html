1. 数据结构在学什么

数据结构这门课的主线可以概括为两句话：

第一，如何用程序代码把现实世界的问题信息化。
第二，如何让计算机更高效地处理这些信息。

现实世界中的金钱、排队、社交关系、文件目录、地图路径，都可以抽象成计算机能处理的数据。数据结构负责设计“数据怎么组织”，算法负责设计“数据怎么处理”。

程序 = 数据结构 + 算法。
数据结构偏向“信息如何表示”，算法偏向“信息如何被处理”。

和其他计算机基础课的关系

课程 核心问题 和数据结构的关系
C 语言 怎么写程序 数据结构常用 C 语言实现，尤其依赖指针、结构体、数组。
数据结构 数据怎么组织，怎么高效处理 上层应用开发的核心基础。
计算机组成原理 硬件如何工作 解释 CPU、内存等底层基础。
操作系统 系统如何管理硬件和程序 解释进程、内存、文件等资源管理。
计算机网络 计算机之间如何通信 解释信息如何跨设备传递。

2. 数据结构的基本概念

数据 信息的载体，是能输入计算机并被程序识别、存储、处理的符号集合。计算机底层最终以二进制形式处理数据。
数据元素 数据的基本单位，通常作为一个整体处理。比如海底捞排队系统中的“一桌顾客信息”。
数据项 构成数据元素的最小单位。比如排队号、取号时间、就餐人数。
组合项 多个数据项组成的一个更大的项。比如生日可拆成年、月、日。
数据对象 性质相同的数据元素的集合。它强调“同类”，不强调元素之间一定有关系。
数据结构 相互之间存在一种或多种特定关系的数据元素的集合。它强调“元素之间的关系”。
例子：某门店正在排队的顾客信息之间有先后关系，所以构成一个数据结构；所有门店的排队顾客信息性质相同，可以看作一个数据对象。
易错区分：数据对象只要求元素性质相同；数据结构还要求元素之间存在某种关系。

3. 数据结构的三要素

学习任何一种数据结构，都要从三个角度看：

逻辑结构 存储结构 数据的运算

3.1 逻辑结构：元素之间是什么关系

逻辑结构 关系特点 例子
集合结构 元素同属于一个集合，除此之外没有其他关系。 烤盘上的食物。
线性结构 一对一关系。除第一个元素外，每个元素有唯一前驱；除最后一个元素外，每个元素有唯一后继。 队列、线性表、排队顾客。
树形结构 一对多关系。 文件目录、思维导图、组织架构。
图状结构 / 网状结构 多对多关系。 微信好友关系、微博关注关系、地图道路。
后续课程重点讨论线性结构、树形结构和图状结构；集合结构一般不是数据结构课程的重点。

3.2 存储结构：逻辑关系如何放进内存

存储结构 核心思想 特点
顺序存储 逻辑上相邻，物理位置也相邻。 需要连续空间；查找第 i 个元素方便；插入、删除可能需要移动大量元素。
链式存储 逻辑关系由指针表示，物理位置可以不相邻。 分配空间更灵活；查找第 i 个元素通常要沿指针走。
索引存储 额外建立索引表，记录关键字和地址等信息。 会额外消耗索引表空间；可提高某些查找效率。
散列存储 / 哈希存储 根据关键字直接计算存储地址。 第六章散列表会详细学习；绪论阶段只需有印象。
顺序存储要求物理连续；非顺序存储中的元素物理位置可以离散。存储结构会影响空间分配方便程度，也会影响查找、插入、删除等运算速度。

3.3 数据的运算：对数据能做什么

运算通常针对逻辑结构定义，针对存储结构实现。

队列的逻辑结构决定它需要“入队”“出队”等操作；如果用顺序存储实现，入队可能是放到数组尾部；如果用链式存储实现，入队可能是申请一个新结点并修改指针。

4. 数据类型与抽象数据类型

数据类型

数据类型是一个值的集合，以及定义在这个集合上的一组操作的总称。

类型 含义 例子
原子类型 值不可再分。 bool、int、char。
结构类型 值可以再分成若干分量。 C 语言中的 struct。

抽象数据类型 ADT

ADT 是抽象的数据组织以及与之相关的操作。它只关心逻辑结构和运算，不关心具体如何存储。

ADT = 逻辑结构 + 数据运算。
真正写代码实现时，才需要选择顺序存储、链式存储、索引存储或散列存储。

5. 算法的基本概念

算法是对特定问题求解步骤的一种描述，是指令的有限序列，每条指令表示一个或多个操作。

做番茄炒蛋时，食材相当于数据结构，菜谱步骤相当于算法。没有食材，无法加工；只有食材没有步骤，也不能得到菜。

算法的五个特性

特性 解释 易错提醒
有穷性 算法必须在有限步后结束，每一步也能在有限时间内完成。 死循环不是算法。程序可以长期运行，但算法必须有穷。
确定性 每条指令含义明确；相同输入必须得到相同输出。 排序时若两个元素关键字相同，也要规定清楚谁在前。
可行性 算法描述的操作可以通过已经实现的基本运算执行。 不能只是想法，要能落成可执行步骤。
输入 算法有零个或多个输入。 Hello World 可以没有输入。
输出 算法至少有一个或多个输出。 没有结果的过程不能称为完整算法。

好算法的目标

正确性 能正确解决问题。
可读性 代码或伪代码容易理解，必要时写注释。
健壮性 面对非法输入能适当处理，不产生莫名其妙的结果。
高效率与低存储需求 时间开销小，空间开销小，也就是时间复杂度和空间复杂度尽量低。

6. 算法的时间复杂度

为什么不用实际运行时间评价算法

  • 运行时间受机器性能影响。
  • 运行时间受编程语言影响。
  • 运行时间受编译器和机器指令质量影响。
  • 有些算法不能先试运行再统计，比如控制系统、实时系统。
时间复杂度研究的是：算法时间开销 T(n) 与问题规模 n 之间的增长关系。

大 O 表示法

时间复杂度只关心数量级：保留最高阶项，去掉常数系数，再套上 O。

T(n) = 3n + 3        => O(n)
T(n) = n^2 + 6n + 7 => O(n^2)
T(n) = 4n^3 + 2n^2 => O(n^3)
规则 含义 例子
加法规则 多项相加，只保留最高阶。 O(n3) + O(n2log n) = O(n3)
乘法规则 嵌套或相乘时，数量级相乘。 O(n) × O(n) = O(n2)

常见数量级

从小到大记：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)。
口诀：常、对、幂、指、阶。常数级 < 对数级 < 幂函数级 < 指数级 < 阶乘级。

怎么数时间复杂度

  1. 找到基本操作，通常是最深层循环里反复执行的语句。
  2. 用问题规模 n 表示基本操作的执行次数。
  3. 取数量级，得到时间复杂度。
// 线性循环：执行 n 次，时间复杂度 O(n)
int i = 1;
while (i <= n) {
    i++;
}

// 两层嵌套循环：内层共执行 n*n 次，时间复杂度 O(n^2)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        // 基本操作
    }
}

// 每次翻倍：循环次数约为 log2 n，时间复杂度 O(log n)
int i = 1;
while (i <= n) {
    i *= 2;
}

最好、最坏、平均时间复杂度

有些算法的执行时间和输入数据本身有关。线性查找就是典型例子。

情况 目标元素位置 复杂度
最好情况 第 1 个位置 O(1)
最坏情况 最后 1 个位置 O(n)
平均情况 假设出现在每个位置概率相同 平均查找次数为 (n+1)/2，复杂度 O(n)
考研分析算法时，通常更关注最坏时间复杂度和平均时间复杂度；最好情况参考价值较低。

7. 算法的空间复杂度

空间复杂度用于衡量算法运行时的内存空间开销与问题规模 n 之间的关系，常用 S(n) 表示。

程序运行需要哪些内存

  • 程序代码：编译后的机器指令，大小通常和问题规模 n 无关。
  • 固定变量、参数、局部变量：如果数量固定，通常只贡献 O(1)。
  • 数组、动态申请的空间：如果大小和 n 有关，要重点分析。
  • 递归调用栈：每一层递归调用都需要保存参数、局部变量、返回地址等信息。

普通程序的空间复杂度

// 只使用固定数量变量，空间复杂度 O(1)
int i;
int sum;

// 使用长度为 n 的数组，空间复杂度 O(n)
int a[n];

// 使用 n*n 的二维数组，空间复杂度 O(n^2)
int matrix[n][n];
如果算法除输入本身外，只使用常数个额外变量，空间复杂度是 O(1)，也称可以原地工作。

递归程序的空间复杂度

大多数考研题中，如果每层递归只使用常数空间，那么空间复杂度主要看递归深度。

// 递归深度为 n，每层只用常数空间，空间复杂度 O(n)
void f(int n) {
    int a, b, c;
    if (n > 1) {
        f(n - 1);
    }
}

// 若第 n 层、n-1 层、...、1 层分别申请不同长度数组，可能达到 O(n^2)
void g(int n) {
    int flag[n];
    if (n > 1) {
        g(n - 1);
    }
}
易错点：递归不只看代码里有没有数组，还要看“递归深度”和“每一层调用需要多少空间”。

8. 考前检查清单

概念题 能区分数据、数据元素、数据项、数据对象、数据结构。
三要素 能说出逻辑结构、存储结构、数据运算，并解释三者关系。
逻辑结构 能判断集合、线性、树形、图状四类结构。
存储结构 能区分顺序、链式、索引、散列，并说出顺序与非顺序的区别。
ADT 知道 ADT 只关心逻辑结构和运算，不关心具体存储实现。
算法 能背出算法五特性和好算法四目标。
时间复杂度 会找基本操作，会看最深层循环，会比较常见数量级。
空间复杂度 会分析数组空间和递归调用栈空间。
本章一句话总复习
数据结构研究数据元素之间的关系以及如何存储和操作这些数据；算法研究解决问题的有限步骤；复杂度研究算法运行时随问题规模增长的时间和空间开销。
