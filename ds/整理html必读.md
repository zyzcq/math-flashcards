六、数据结构图示要求
==================================================

复杂的地方必须用“代码绘图”展示。

优先使用 Mermaid，但也可以使用 HTML/CSS/SVG。

请根据内容自动选择合适图形。

1. 顺序表
   - 用数组格子展示。
   - 标出下标、位序、length、capacity。
   - 插入时展示元素后移。
   - 删除时展示元素前移。

2. 单链表
   - 用结点框展示 data 和 next。
   - 用箭头表示 next。
   - 标出 head、p、q、s 等指针。
   - 插入删除时展示指针修改前后。

3. 双链表
   - 展示 prior 和 next 双向箭头。
   - 插入删除时标出四条指针如何变化。

4. 循环链表
   - 展示尾结点指回头结点。
   - 空表时展示 head 指向自己。
   - 遍历时标出终止条件不是 NULL，而是回到 head。

5. 静态链表
   - 用表格展示数组下标、data、next。
   - 用箭头展示逻辑链。
   - 标出 -1 表示结束，-2 表示空闲。
   - 强调位序和数组下标不同。

6. 栈
   - 用竖直栈图展示 top。
   - 展示入栈、出栈前后。
   - 顺序栈和链栈分别表示。

7. 队列
   - 用横向数组或链式结构展示 front、rear。
   - 循环队列要展示取模。
   - 标出队空、队满条件。

8. 树和二叉树
   - 用树形图展示根、孩子、叶子。
   - 遍历时可用流程图或序列展示。
   - 递归过程可以用调用栈图展示。

9. 图
   - 使用 Mermaid graph 展示顶点和边。
   - BFS 用队列过程图。
   - DFS 用递归栈或访问路径图。
   - 最小生成树、最短路径等要展示过程。

10. 排序
   - 用数组状态变化展示每一趟。
   - 复杂排序可以用流程图和表格结合。
   - 例如快速排序标出 pivot、left、right。
   - 堆排序用堆树和数组对应关系。

11. 查找
   - 顺序查找用扫描过程图。
   - 折半查找用 left、mid、right 图。
   - 哈希表用桶和冲突链展示。
   - B 树/B+ 树用树图展示关键字分布。

如果 Mermaid 不适合表达某个图，请用 HTML/CSS 或 SVG 自己画。

10. 代码块后可以添加“关键语句解释”。

例如，对于链表插入代码：

s->next = p->next;
p->next = s;

需要解释：
- 第一行：让新结点先接上原来的后继。
- 第二行：让前驱结点指向新结点。
- 顺序不能反，否则会丢失原来的后继。

十四、必须特别注意的数据结构细节
==================================================

如果文稿中出现以下内容，必须重点展示：

1. 头结点和首元结点区别。
2. 位序从 1 开始，数组下标通常从 0 开始。
3. 链表插入删除必须注意指针修改顺序。
4. 循环链表不能用 NULL 作为遍历结束条件。
5. 双链表插入删除涉及 prior 和 next 两个方向。
6. 静态链表中游标不是指针，而是数组下标。
7. 顺序表插入删除需要移动元素。
8. 链表插入删除如果已知前驱是 O(1)，如果要查找前驱通常是 O(n)。
9. 顺序表支持随机存取，链表不支持。
10. malloc 申请的空间必须 free。
11. 队列中 front、rear 的含义要说清。
12. 循环队列牺牲一个单元判满时，队满条件要准确。
13. 栈和队列的操作受限特点要突出。
14. 树的递归定义和遍历顺序要清晰。
15. 图的 BFS 和 DFS 辅助结构不同。
16. 排序算法的稳定性、时间复杂度、空间复杂度要保留。
17. 查找算法的适用前提要保留。
